WHAT IS IN SCOPE FOR FINDINGS

Category A — JAPANESE LANGUAGE ACCURACY (CRITICAL severity)
-----------------------------------------------------------

A1. Incorrect grammar
    - Register stacks: e.g. 「たかいなあですね」 (cannot combine なあ
      casual + です polite + ね). Each register-marker drops on the
      stack.
    - Conjugation errors: ru-verb tagged as u-verb (or vice versa),
      irregular verb (来る / する / 行く) miscategorised.
    - Wrong copula: な-adjective + だ vs です stacking (「きれいだで
      す」 ✗ → 「きれいです」 ✓).

A2. Unnatural collocations
    - Object-particle for 好き / 嫌い / 上手 / 下手 / ほしい / わかる:
      が-particle is the N5-canonical default; を only in specific
      lexicalisations a native catches.
    - て-form chains that mix unrelated actions where a comma /
      separate-sentence would be more natural.
    - Counter-noun collocations (本 for cylinders, 枚 for flat
      objects, 匹 for small animals — register-correct N5 mappings).

A3. Wrong particle
    - は / が default-vs-contrast at N5 level (we test this; we cannot
      ship examples that get it wrong).
    - に vs で for location: に for existence/arrival; で for action-
      location.
    - に vs へ for direction: both work for movement; へ has a
      directional nuance, に has a destination nuance.
    - を vs が with stative verbs (見える, 聞こえる take が; 見る,
      聞く take を).

A4. Wrong reading
    - On-yomi vs kun-yomi assignment per kanji.
    - 一日 = ついたち (1st of month) vs いちにち (one day) — context
      decides.
    - Compound-rendaku triggers (1/6/8/10 → 〜ぴき, not 〜ひき).
    - Sokuon (small ッ) listed as a separate kun-reading rather than
      as a phonological assimilation (e.g. みっつ / よっつ / むっつ /
      やっつ ARE sokuon-allophones of み / よ / む / や, NOT distinct
      readings).

A5. Wrong kanji usage
    - Homograph confusion: 神 (かみ god) vs 髪 (かみ hair) vs 紙 (かみ
      paper) — same reading, different kanji.
    - 上手 (じょうず "skilful") vs 上手 (うわて "upper hand /
      upstream") — same kanji, different reading.
    - 行 covers いく / おこなう — both N5 readings, context-decided.

A6. Onyomi / kunyomi misclassification
    - Sino-Japanese readings (on) vs native Japanese readings (kun)
      mixed up in the kanji.json on/kun fields.
    - Borderline cases (e.g. 茶 only has on-yomi cha/sa at N5; flag
      if a kun-reading is invented).

A7. Vocabulary meaning errors
    - Gloss_en or gloss_hi semantically wrong (e.g. すずしい glossed
      as "cold" — it's "cool" / "pleasantly cool", not 寒い).
    - Polysemy collapse: 上 has multiple senses; the gloss must list
      the senses N5 learners need, not all 6 dictionary senses.
    - Over-narrow gloss: e.g. かける glossed only as "to hang" when
      the N5 corpus uses it for 「めがねを かける」 (to wear glasses)
      and 「でんわを かける」 (to make a phone call).

A8. Mistranslation of Japanese → English
    - English gloss adds a nuance the Japanese doesn't have, or vice
      versa. Test: does the back-translation match?
    - Causative / passive confusion in glosses.

A9. Half-applied ウ音便 in keigo (politeness_ladder.humble)
    - i-adjective + ございます forms must complete the contraction:
        a + u → ō:  たかい → たこう (NOT たかう)
                   ありがたい → ありがとう (NOT ありがたう)
                   はやい → はよう (NOT はやう)
        i + u → yū: おいしい → おいしゅう (NOT おいしう)
                   うれしい → うれしゅう
                   やさしい → やさしゅう
        u + u → uu: やすい → やすう (stays — DO NOT contract)
        o + u → ō:  irregular (e.g. 良う / 無う)
    - Half-applied forms read as broken to natives. Reference cases:
      §3.2.33 in the procedure manual.

A10. Sokuon / rendaku as separate readings
     - 撥音便 / 促音便 outputs (e.g. 一本 いっぽん / 一杯 いっぱい)
       are derived; do not list them as base readings in kanji.json.
     - Rendaku-shifted readings (e.g. 子供 こども vs 三日 みっか)
       are derived; do not list them as separate kun.

A11. Mixed orthography for the same morpheme
     - Pick one convention per surface and stick with it. Examples
       that fail: 時はん vs 時半 in different example sentences for
       the same pattern; ひと vs 人 mixed across the same kanji's
       example list.
     - Bunsetsu-spaced kana vs unspaced (e.g. 「あした がっこうに
       いきます」 vs 「あしたがっこうにいきます」 — pick one for the
       N5 listening / dokkai surface; document the choice).

A12. meaning_ja CROSS-CONTAMINATION (added 2026-05-13; closed by JA-75)
     A failure mode caught 28 times across 3 native-teacher audit
     runs before JA-75 locked it down. The `meaning_ja` field's
     leading 「marker」 must reference the SAME grammar point as the
     entry's `pattern` field. Catastrophic failure mode: meaning_ja
     describes a completely different rule.
     Examples caught (now fixed; documented for reference):
       - n5-110 (counter pattern) had meaning_ja about 〜たい
       - n5-126 (が) had meaning_ja about 〜から、〜
       - n5-142 (〜にします) had meaning_ja about な-Adj+な+めいし
       - n5-154 (もう+ました) had meaning_ja about へ direction
       - n5-183 (Question word + か/も) had meaning_ja about Verb-た+ことがある

     CURRENT GUARDS (in order of strength):
       1. JA-75 (PRIMARY — installed 2026-05-13): per-pattern
          `_meaning_ja_markers` list (internal metadata, snapshot of
          the verified-correct meaning_ja's distinctive vocabulary).
          meaning_ja must contain at least 2 markers, OR at least 1
          distinctive (>=3-char) marker. Catches future drift AWAY
          from the verified state — including subtle 1-char-overlap
          cases that JA-71 missed.
       2. JA-71 (SECONDARY): character-overlap heuristic. Still
          green; serves as a coarse first-line check. Limitation:
          single-char incidental overlap (も, が, で) lets contamination
          pass — that's why JA-75 supersedes it.

     What JA-75 STILL cannot catch:
       - Edits that PRESERVE the marker vocabulary but reword the
         actual meaning (rare; would require LLM semantic comparison).
       - Authoring of a brand-new pattern where the markers themselves
         get authored incorrectly from the start.
     For these, manual native-teacher sampling remains the catch-all.

A13. PITCH-ACCENT MORA COUNT ERRORS (added 2026-05-13)
     Tokyo NHK convention: each kana = 1 mora, EXCEPT small kana
     (ゃゅょぁぃぅぇぉ) merge with the preceding mora; ー and っ are
     each 1 mora. Common failure modes from LLM-authored entries:
       - げんき (3 mora, ge-n-ki) listed as mora=1
       - しずか (3 mora, shi-zu-ka) listed as mora=1
       - 先週 せんしゅう (4 mora) listed as mora=1
       - おにいさん (5 mora, o-ni-i-sa-n) listed as mora=4
       - おいしい (4 mora, o-i-shi-i) listed as mora=3
     Verify pitch_accent.mora == count_mora(reading) by hand for any
     sampled entry. The kanjium dictionary at
     not-required/tools-archive/_cache_kanjium_accents.txt has the
     canonical values for all 1009 N5 entries. CI invariant JA-70
     locks this forward.

A14. PLACEHOLDER / SELF-INCRIMINATING TEXT IN SHIPPED DATA (added 2026-05-13)
     Look for text that admits the entry is incomplete or wrong:
       - "(Fallback ref:)" prefix in public_domain_refs.context
       - "PD pending until 20XX" in pd_status (with year >= 2026)
       - "TODO" / "FIXME" / "see above" / "placeholder" anywhere in
         user-visible fields
       - "(This pattern is also indexed as 'X')" parentheticals in
         explanation_en (use structured see_also field instead)
     These betray that the entry was authored as a placeholder and
     never replaced. CI invariant JA-69 catches the legal-status
     version (pd_status admissions of in-copyright).

       "じぶんで しゅくだいを します。"      in 14 patterns
       "あにより わたしのほうが はやく ..."  in 14 patterns
       "父に とけいを もらいました。"        in 11 patterns
     ...etc.

     Pattern: pick a random grammar pattern with `tier: late_n5` or
     a conjugation-form pattern (Verb-X, い-Adj-X, な-Adj-X) and
     check whether indices [4][5][6] contain examples that USE the
     target pattern's marker. Examples to verify against:

       n5-044 こう/そう/ああ/どう → [4][5][6] previously held
         てください request sentences (くちを あけて ください, etc.)
         that don't demonstrate manner-demonstratives.

       n5-104 Verb-stem + たいです → [5][6] previously held
         "なぜ来なかったのですか" / "しけんはむずかしかったです" —
         neither uses たいです.

       n5-167 〜んです/〜のです → [4][5][6] previously held plain
         です sentences (no んです at all).

     Output format for findings:
       Pattern: <n5-XXX>
       Index: [N]
       Issue: example does not demonstrate the pattern's marker
         (target marker: "<...>", example: "<ja>")
       Fix proposal: <new sentence using the marker, preserving form tag>

     CI gate: JA-81 (boilerplate-leak guard at threshold 10
     patterns) already catches the extreme cases. The mid-range
     (3-9 occurrences) requires this manual audit.

A16. FORM-TAG VALUE MISMATCHED WITH EXAMPLE CONTENT
     (added 2026-05-14)

     Every example object has a `form` field whose value is one of
     {affirmative, past, negative, question, habitual, ...} OR a
     pattern-specific role tag (e.g. "weather-agreement",
     "exclamation-beautiful"). The value must MATCH the example's
     grammatical form:

       form="past" example MUST contain past marker (〜た, 〜ました,
         〜でした, 〜かった, 〜だった, 〜んだ)
       form="negative" example MUST contain negative marker (〜ない,
         〜ません, 〜じゃない, 〜では ない, 〜なかった, 〜なくて)
       form="question" example MUST end with か / ？ / の / よね /
         かな / だろう / でしょう

     Common mismatch patterns:
       - form="question" but example is a の-final informal Q tagged
         as affirmative (or vice versa)
       - form="past" tagged on a negative-non-past example (e.g.
         "じゃ ありません" tagged past — that's present-negative)
       - form="affirmative" tagged on an opinion-question ending in
         ですか

     False positive to skip: relative-clause past markers appear
     mid-sentence (e.g. "わたしが 買った 本" / "I bought the book")
     — the た is on the verb modifying the noun, not at sentence
     end. Don't flag these.

A17. WITHIN-PATTERN DUPLICATE EXAMPLES
     (added 2026-05-14)

     Same `ja` string appearing at two different indices in one
     pattern wastes a slot. Almost always introduced by a content-
     authoring pass that didn't check sibling examples before
     authoring a new one.

     Detection: for each pattern, build a dict of ja → index. Any
     ja value seen twice is a duplicate.

     Fix policy: keep the first occurrence; replace the second with
     a newly-authored sentence demonstrating the same pattern + same
     form tag, using N5-vocab + N5-kanji only.

A18. VOCAB EXAMPLE TEMPLATE-LEAK: 'Xを 見ました。'
     (added 2026-05-15; locked by JA-83)

     Anti-pattern: data/vocab.json carried 108 entries whose example
     at index [1] was the literal template 'Xを 見ました。' →
     "I saw X." The Japanese is grammatical but semantically weak
     for many headwords:
       - 'おとうとを 見ました。'   "I saw younger brother." (unnatural)
       - 'けさを 見ました。'       "I saw this morning." (nonsense)
       - 'なつを 見ました。'       "I saw summer." (nonsense)
       - 'となりを 見ました。'     "I saw next to." (nonsense)
       - 'まいばんを 見ました。'   "I saw every night." (nonsense)
     The English translations are also broken — most omit the
     article ("I saw wall.", "I saw pork."), confirming bulk LLM
     generation without review.

     Detection regex (strict; require X is a single token, no
     whitespace, so legitimate sentences like 'どうぶつえんで ぞうを
     見ました。' / "I saw an elephant at the zoo" pass through):

         ^[^\s]{1,10}を\s*見ました(?:よ|ね|よね)?。?$

     Fix policy: replace with a natural N5-level sentence that uses
     the headword in a different semantic frame
     (e.g. Xを 買いました / Xが すきです / Xに 行きます / Xを つかいます).

     CI gate: JA-83 prevents future regression.

A19. VOCAB EXAMPLE TEMPLATE-LEAK: 'あの Xは どこですか。' for
     non-location headwords
     (added 2026-05-15; locked by JA-83)

     Anti-pattern: 111 entries used 'あの Xは どこですか。' →
     "Where is that X?" at index [1] where X is a time word, body
     part, abstract noun, or other non-location concept:
       - 'あの 九月は どこですか。'       "Where is that September?"
       - 'あの ときは どこですか。'        "Where is that time?"
       - 'あの おとといは どこですか。'    "Where is that day before yesterday?"
       - 'あの けいかんは どこですか。'    "Where is that police officer?"
     Locational `どこ` is appropriate for places (the trash entry
     for ホテル would naturally read 'ホテルは どこですか。'). For
     non-location headwords, the question is nonsense.

     Detection (skip when entry.section starts with 13-locations
     or 26-house):

         pat = re.compile(r"^あの\s+(.{1,10})は\s*どこですか。?$")
         m = pat.fullmatch(ja)
         if m and not is_locationish and m.group(1).strip() == form.strip():
             flag

     Fix policy: replace with a natural sentence using the headword
     in context appropriate to its semantic class (time → 'Xに 〜',
     body part → 'Xが いたいです', person → 'Xは 〜さんです', etc.).

A20. VOCAB EXAMPLE TEMPLATE-LEAK: 'これは Xです。' / 'あれは Xです。'
     bare introduction on entries with 3+ examples
     (added 2026-05-15; locked by JA-83)

     Anti-pattern: 178 noun entries used bare 'これは Xです。' /
     'あれは Xです。' / 'それは Xです。' at example[1] where the
     entry already has 3 or more examples. These bare templates
     waste an example slot:
       - 'これは お母さんです。'   "This is mother." (no article in EN)
       - 'あれは 男です。'         "That is man." (bare)
       - 'これは えです。'          "This is picture." (bland)
     The template is fine for an INTRODUCTORY example; it becomes
     filler when the entry has 3+ examples and other slots already
     demonstrate the headword.

     EXCEPTION: for the demonstrative-pronoun entries themselves
     (これ / それ / あれ / この / その / あの), 'これは Xです。' IS
     the canonical demonstration of the headword. Do NOT flag for
     entries with section/pos in the 5-demonstratives family.

     Detection:

         pat = re.compile(r"^(これ|あれ|それ)は\s*(.{1,10})です。?$")
         is_demonstrative = (entry.pos == "demonstrative" or
                             "demonstrative" in entry.section.lower())
         if pat.fullmatch(ja) and not is_demonstrative \
            and len(entry.examples) >= 3:
             flag

     Fix policy: replace with a natural usage of the headword
     (e.g. 'Xは どこですか。', 'Xを 〜', 'Xが 好きです。').

A21. VOCAB EXAMPLE TEMPLATE-LEAK: '「X」と あいさつしました。' where
     X is not a greeting
     (added 2026-05-15; locked by JA-83)

     Anti-pattern: 12 entries in sections 36-greetings and 39-fillers
     used '「X」と あいさつしました。' → "I greeted with X." even when
     X was a non-greeting filler:
       - '「えーと」と あいさつしました。'   (えーと = "um/uh")
       - '「いいえ」と あいさつしました。'   (no isn't a greeting)
       - '「ええ」と あいさつしました。'     (yes isn't a greeting)
       - '「そうですか」と あいさつしました。' (response, not greeting)

     Detection:

         pat = re.compile(r"^「(.{1,10})」と\s*あいさつしました。?$")
         GREETINGS = {"おはよう","こんにちは","こんばんは","ありがとう",
                      "さようなら","おやすみ","ただいま","おかえり",
                      "もしもし","いってきます","いってらっしゃい",
                      "おかげさまで"}
         m = pat.fullmatch(ja)
         if m and m.group(1) not in GREETINGS:
             flag

     Fix policy: change the verb to 言いました or restructure into a
     more natural usage (e.g. '「えーと」と 聞きました。', '「いいえ、
     ちがいます」と 言いました。').

A22. VOCAB CROSS-ENTRY BOILERPLATE (the JA-81 analogue for vocab)
     (added 2026-05-15)

     A single ja sentence appearing as an example in 5 or more vocab
     entries is suspicious. Some reuse is legitimate when each entry
     teaches a different word in the same sentence — e.g. the
     sentence 'コーヒーは 飲みますが、にくは たべません。' naturally
     appears under コーヒー, にく, 飲む, 食べる. But the threshold for
     "natural reuse" is much lower than for grammar (where JA-81 uses
     10). When the count hits ≥ 5, audit each occurrence:
       - KEEP if the entry's headword is genuinely demonstrated by
         that part of the sentence.
       - REPLACE if the entry's headword is incidental (the sentence
         could have been written without it). Example: the counter
         まい entry using 'まいにち にほんごを べんきょうします。' —
         that's 毎日, not the counter 枚.

     Threshold: ≥ 5 → audit. The 2026-05-15 pass diversified 5
     sentences (8x→4x, 7x→2x, three 5x→3-4x) using this rule.

A23. VOCAB HEADWORD-MISSING-FROM-EXAMPLE
     (added 2026-05-15)

     Every example sentence must use its entry's headword somehow.
     Examples that don't contain the headword are silent bugs: the
     learner sees the example but doesn't see the word being taught.

     Detection: for each example, check that ja contains AT LEAST
     ONE of:
       - the entry's `form`
       - the entry's `reading`
       - any string in entry.readings[] (for multi-reading entries)
       - a verb stem (for verb entries) — see A23a
       - a counter-rendaku alternant (for counter entries) — see A23b

     Failure mode: 'たばこ' entry had example 'バスで 学校へ 行きます。'
     at [1] — about bus/school, no たばこ at all. Real bug.

     A23a. Verb-stem awareness (so まつ→まちます counts as a hit):
            verb-1 (godan): drop the final u-row kana
              まつ → ま; かう → か; よむ → よ; はたらく → はたら
            verb-2 (ichidan): drop the final る
              たべる → たべ; おきる → おき; ねる → ね
            verb-3 (irregular):
              する → し / す ; くる → こ / き / く
              -する compounds: べんきょうする → べんきょう
            Compound -くる verbs (e.g. もってくる, つれてくる):
              prefix + {き, こ, く} all count as headword stems.

     A23b. Counter rendaku (so 三びき counts as a ひき hit):
             ひき → {ぴき, びき}     (一ぴき, 三びき, 八ぴき)
             はい → {ぱい, ばい}     (一ぱい, 三ばい)
             ふん → {ぷん}          (一ぷん, 三ぷん)
             ほん → {ぽん, ぼん}     (一ぽん, 三ぼん)
             ほ   → {ぽ, ぼ}
             かい → {がい}          (三がい floor counter)
             けん → {げん}

     Fix policy: replace the example with one that uses the headword
     in inflected or rendaku-altered form.

A37. COUNTER-NOUN SEMANTIC PAIRING
     (added 2026-05-15 native-teacher pass; locked by JA-89 lock1)

     Even when the counter is grammatically possible, examples that
     pair a noun with a counter from the WRONG canonical set teach
     learners the wrong pairing. JLPT N5 establishes canonical
     pairings:

       本 (ほん/ぼん/ぽん) → cylindrical objects: pens, bottles,
                            umbrellas, trees
       枚 (まい)          → flat thin objects: paper, photos,
                            tickets, plates, shirts
       冊 (さつ)          → bound books / magazines / notebooks
       個 (こ)            → small / general 3D objects: apples,
                            eggs, balls
       人 (にん, ひとり/ふたり) → people
       匹 (ひき/ぴき/びき) → small animals: dogs, cats, fish (live),
                            insects
       頭 (とう)          → large animals: horses, cows, elephants
       台 (だい)          → machines / vehicles
       杯 (はい/ぱい/ばい) → cupfuls / glasses of liquid
       軒 (けん, N4)      → buildings / houses (defer)
       足 (そく)          → pairs of footwear (shoes, socks)
       つ (native count)  → general fall-back; OK for ~10 objects

     Anti-pattern (caught 6 instances in 2026-05-15 pass): native-
     counter-series teaching entries 三つ..九つ used 「本が Xつ
     あります」. While grammatically possible, books canonically
     take 冊. Replace with a noun that's natively つ-paired (りんご /
     ボール / いす / コップ / はこ).

     Detection: for each `<num><counter>` token, walk backwards to
     the noun bound by particle (を/が/に/も). Look up the noun in
     canonical-pairing table. Flag if counter ∉ valid set for that
     noun.

     CRITICAL false-positive avoidance:
       - 倍 (ばい) is a multiplier suffix (三倍 = "triple"), NOT a
         noun-counter. Exclude from the scan.
       - 「Xの Yを N counter」 — the closer noun (Y) is the counted
         one, not the modifier (X). Use rightmost match.

     Fix policy: replace the example noun with a canonically-paired
     one OR change the counter to the canonical one for that noun.

A38. ANIMACY VIOLATIONS: ANIMATE NOUN + が あります
     (added 2026-05-15 native-teacher pass; locked by JA-89 lock3)

     Animate nouns (people / animals) must use います for existence.
     Inanimate nouns use あります. Violations:
       - 男の子が あります        → must be 男の子が います
       - 日本人が あります        → must be 日本人が います
       - スペイン人が あります    → must be スペイン人が います
       - そぼが あります          → must be そぼが います

     Detection: maintain an ANIMATE_NOUNS list. For each sentence,
     scan for `<animate-noun>(?:は|が)\s*あります`.

     CRITICAL false-positive avoidance: substring matches. The
     pattern 「ぼうしが あります」 (hat — inanimate) must NOT match
     because うし (cow, animate) is a substring of ぼうし. Use
     `(?<![ぁ-ゟ一-鿿])<noun>` lookbehind to require word-start.

     (added 2026-05-15 native-teacher pass)

     Vocab examples whose ja text doesn't contain the entry's
     headword at all are silent bugs (the learner sees an example
     but doesn't see the word being taught). The 2026-05-15 audit
     caught 1 new such bug:
       - すう entry [1]: 「バスで 学校へ 行きます。」 (about bus +
         school; no smoking content). Same bug class as たばこ
         caught in earlier audit.

     Detection requires:
       - Allowing inflected verb forms (まつ → まちます; する →
         します/して/しよう; くる → きます/きて/こない)
       - Allowing inflected i-adjective forms (おもしろい →
         おもしろくて / おもしろかった / おもしろくない)
       - Allowing counter-rendaku alternants (ひき → びき/ぴき)
       - Skip when the entry's kanji-form character appears in
         the ja (e.g., 学 in 学校)

A41. UNNATURAL ADVERB-VERB PAIRINGS
     (added 2026-05-15 native-teacher pass)

     Some adverbs require specific verb-mood pairings:
       - ぜひ + volitional / request / want: ぜひ 来てください ✓,
         ぜひ 食べたい ✓ — but NOT plain ぜひ 食べます (the adverb
         expects choice or desire, not flat declaration).
       - ただ + casual / minimization: ただの 水 ✓, ただ 見るだけ ✓
         — but NOT bare ただ ごはんを 食べます (ambiguous between
         "just/only" and "free", semantically thin).
       - やはり + opinion-confirmation: やはり 〜が 好きです ✓ —
         can pair with neutral declarative but feels heavy without
         contrast context.

     Caught at the vocab.adverb entries' example[1] slot. Fix by
     pairing the adverb with the right verb mood.

A42. POTENTIAL/EXISTENTIAL VERB + VOLITIONAL つもり MISMATCH
     (added 2026-05-15 native-teacher pass)

     つもり ("intend / plan to") attaches to volitional verbs (verbs
     of intentional action). It does NOT pair with:
       - 聞こえる (potential / spontaneous "be audible") — caught
         at vocab 聞こえる[1]: 「あした 聞こえるつもりです」 is
         "I plan to be audible tomorrow" — nonsense.
       - ござる (existential keigo, equivalent to ある) — caught
         at vocab ござる[1]: 「あした ござるつもりです」 is "I plan
         to exist tomorrow" — nonsense.
       - Intransitive change-of-state verbs like はじまる / しまる /
         おちる (caught at the template-leak in A43 below).

     Native judgment: any X-つもり where X is non-volitional /
     intransitive / spontaneous is wrong.

A43. VOCAB VERB-TEMPLATE LEAKS: '毎日 X ことが できます' and
     'あした X つもりです'
     (added 2026-05-15 native-teacher pass; locked by JA-89 lock5)

     Two systemic templates at vocab[verb-entry].examples[1]:

       '毎日 VERB-dictionary ことが できます。'
         = "I can VERB every day" — leaked into 28 verb entries.
         Often semantically nonsensical for non-volitional verbs:
           - 毎日 こまる ことが できます  ("can be troubled daily")
           - 毎日 くもる ことが できます  ("can become cloudy daily")
           - 毎日 おちる ことが できます  ("can fall daily")

       'あした VERB-dictionary つもりです。'
         = "I plan to VERB tomorrow" — leaked into 30 verb entries.
         Several use intransitive change-of-state verbs that don't
         take volitional つもり (see A42):
           - あした はじまる つもりです  (intransitive)
           - あした しまる つもりです     (intransitive)
           - あした なくす つもりです     (involuntary loss)

     Detection regex:
       r"^毎日\s+.*ことが\s+できます。?$"
       r"^あした\s+.*つもりです。?$"

     Fix policy: replace with a natural usage of the verb that
     respects its volitionality / transitivity class.

A44. MULTI-TRANSLATION EN: "X / Y" pick-one
     (added 2026-05-15 native-teacher pass)

     translation_en fields with " / " separating two or more
     alternative English renderings are bug-prone:
       - "I can use Japanese. / I'm able to do (in) Japanese."
       - "I eat meals by all means / definitely."
       - "There is a person who save / accumulate."  (also relative-
         clause template leak)

     Detection: regex r" / " in translation_en, length > 30 chars.

     Fix policy: pick the most natural English; commit to one
     rendering. Translators decide; do not ship optionality to the
     learner.

A45. TEMPLATE 'あの Xは どこですか。' IN GRAMMAR.JSON
     (added 2026-05-15 native-teacher pass; locked by JA-89 lock4)

     The same 'あの Xは どこですか。' template anti-pattern caught in
     vocab.json (see A19) also leaked into grammar.json. 5 instances
     across n5-041, n5-048, n5-164 (X = みなさん / お父さん — both
     non-locations). The earlier audit cleaned vocab but never
     extended the scan to grammar — sister-corpus blindspot.

     Detection identical to A19. CI gate JA-89 (lock4) extends the
     scan to grammar.json.

A46. PITCH ACCENT VALIDATED AGAINST EXTERNAL REFERENCE
     (added 2026-05-15; locked by JA-90)

     The vocab.json `pitch_accent.drop` field stores an integer drop-
     mora position per entry. Drop position is auditory phenomenology
     — LLM training has limited explicit accent annotation and
     confidence on this dimension is genuinely lower than on syntax
     or morphology.

     **Reconciliation methodology (2026-05-15):**

     1. Fetch external reference dataset (kanjium project, CC-BY-SA
        4.0). The dataset compiles from EDICT/EDRDG and NHK 日本語
        発音アクセント新辞典. Pin to a specific upstream commit SHA
        for reproducibility.

     2. Filter to N5 vocab scope by (form, reading). Use a TWO-TIER
        matching approach:
          - Tier 1: exact (form, reading) match
          - Tier 2: reading-only fallback (for N5 entries with kana-
            only forms where kanjium has the kanji-first form)
        Save the filtered subset to `data/n5_pitch_accent_reference.json`
        with full `_meta` block (source, commit SHA, license,
        attribution, regenerate_with command).

     3. Per-entry classification:
          - MATCH: current drop ∈ reference drops → confidence='high'
          - DISAGREE: current drop ∉ reference drops → fix to
            reference[0], confidence='high', log diff
          - NOT_FOUND: vocab not in reference → confidence='unverified',
            keep current value (transparent gap)

     4. Add `pitch_accent.confidence` field to EVERY vocab entry
        (Option C transparency). Values: 'high' | 'medium' | 'low' |
        'unverified'. JA-90 enforces:
          confidence='high' AND drop ∈ reference_drops, OR
          confidence in {'medium', 'low', 'unverified'}

     **N5 2026-05-15 reconciliation results:**
        MATCH:           810/1009  (80%)
        DISAGREE_FIXED:   22/1009  ( 2%)
        NOT_FOUND:       177/1009  (18%) — gairaigo + function words
     Total kanjium-validated: 832/1009 (82%) at confidence='high'.

     **Caveats embedded in the methodology:**
        - Tokyo dialect bias (kanjium reflects standard); Kansai/
          Kyushu accent maps differ entirely
        - by-reading matches use UNION of all drops across kanjium
          entries with that reading — wider but less precise
        - NOT_FOUND entries (mostly loanwords) genuinely lack a
          reference; native-human auditory review remains queued

     **For Nx levels:** apply this same pattern in audit cycle 4-5
     (after structural + linguistic saturation). The refresh tool at
     `tools/refresh_pitch_accent_reference.py` is reusable across
     levels — just point at the level's vocab.json.

Category B — JLPT N5 SCOPE LABELING (HIGH severity)
---------------------------------------------------

B1. N4-or-above grammar treated as N5 without late_n5 / borderline
    flag. Known N4-leakers to specifically check:
    - 〜について, 〜に関して, 〜に対して — N3+ topic markers.
    - 〜ばかり — N4.
    - 〜たら conditional (sequence/result) — N4.
    - 〜ても concession — N4-edge.
    - 〜ながら + relative clause — N4.
    - 〜たり〜たりする with multiple actions — N4 expansion of N5
      core.
    - 〜ようにする / 〜ようになる — N4.
    - Passive 〜られる / causative 〜させる — N4.

B2. N4+ vocab without `tier: late_n5` or `tier: borderline_n4`
    Known leakers (seen in past audits):
    - 紹介する, 卒業, 大学院, 雑誌, 階段, ポケット (some of these
      ARE N5; verify against jlptsensei / Tanos / Marugoto tables).

B3. N4+ kanji without exception-list documentation
    - 田舎, 億, 銀, 黒, 風, 呂, 雑, 誌, 紹, 介, 卒, 業, 舎, 荷 —
      cross-check against data/n5_kanji_whitelist.json.

B4. Example sentences using N4+ kanji or vocab in N5-core patterns
    - Pattern-level tier might be N5, but its example uses 紹介する.
      Flag for vocab swap OR tier downgrade.

B5. Dokkai passages with kanji-density exceeding N5 cap
    - Per-passage kanji-cardinality rule: ≤2 N4-borderline kanji per
      passage, all whitelisted via the JA-28 exception list.

Category C — PEDAGOGICAL QUALITY (MAJOR severity)
-------------------------------------------------

C1. Explanations that confuse N5 learners
    - Too abstract — define by USE, not by linguistic theory at N5.
    - Too many alternatives in one breath — pick the canonical form,
      mention alternates separately.
    - Wrong audience assumption (assuming familiarity with Japanese
      linguistic terminology — 連用形 / 連体形 — at N5).

C2. Missing learner-critical cautions
    - 寝る (ru-verb) vs 寝ない irregular pattern — common L2 trap.
    - 来る / 行く forward/away from speaker confusion in te-form.
    - 〜たい vs 〜たがる person-marking restriction.

C3. Missing distinction between near-synonyms
    - すき vs だいすき (intensity).
    - いい vs よい (register: いい casual, よい slightly more formal).
    - もう / まだ (already / still / not-yet — perfective/imperfective).
    - 〜たい vs ほしい (verb-volition vs noun-desire).

C4. Ambiguous meaning fields
    - あめ (rain / candy) — both N5. Gloss must disambiguate or list
      both with reading context.
    - はし (bridge / chopsticks / edge) — pitch-accent + context.
    - おもい (heavy / think) — different verbs.

C5. Overly broad meanings for beginner kanji
    - 上 has 5+ dictionary senses; N5 needs at most 2-3.
    - 下 same constraint.
    - 行 needs to highlight いく (go) at N5, OK to add おこなう (carry
      out) for N4 — flag if the N4 sense is in the N5 file without
      tier flag.

C6. Inconsistent terminology across files
    - "transitive" vs "vt." vs "他動詞" — pick one convention.
    - "Group 1 / Group 2 / Group 3" vs "u-verb / ru-verb / irregular".

C7. Easy-Japanese (やさしいにほんご) register violations
    When a field is labeled with the 「やさしいにほんご」 register tag
    (typically in `meaning_ja` rendered in the UI as 意味（やさしい
    にほんご）), the content MUST follow the accessibility-tier rules
    (弘前大学 やさしい日本語 guidelines / NHK NEWS WEB EASY register):
    - One idea per sentence; ≤30 characters.
    - N5-floor vocab only — no しつもん (N4), について (N3),
      に関して (N3), 〜ばかり (N4), 〜ばかりでなく.
    - No double negative, no passive voice, no causative.
    - Concrete examples (「あさ ごはんを たべました」), not abstract
      definitions.
    - If kanji used, give furigana / bunsetsu-spaced kana.
    Reference: §3.2.25 in procedure manual.

C8. Pattern-title quality
    - No placeholder POS-only labels (e.g. "Verb", "Adjective + Noun",
      "Particle"). Each pattern title must be specific enough to be
      searchable + memorable.
    - Examples of acceptable titles: 「V-plain + N (relative clause)」,
      「Adj + N (combined)」, 「V-plain + まえに」.

Category D — TEST / QUESTION QUALITY (HIGH severity)
----------------------------------------------------

D1. Questions not at N5 level
    - Stem or choices include N4+ vocab without proper labeling.
    - Reading passage too dense, too long, or too abstract.

D2. Multiple possible correct answers
    - Particularly common with interchangeable particle pairs (に/へ
      for direction, は/が in narrow contexts). Flag any question
      where 2+ choices are pragmatically valid without scene context.

D3. Ambiguous distractors
    - Distractors that overlap semantically with the correct answer
      such that a native couldn't choose definitively.
    - Distractors that are technically wrong for one reason but
      could be argued correct on another.

D4. Wrong correct-answer key
    - The "correctAnswer" field doesn't match the intended target;
      check rationale text vs answer index.

D5. Distractor quality
    - Too easy: all 4 are wildly different POS / category — gives
      the answer away.
    - Too advanced: distractor uses N3+ structure to test an N5
      point.
    - Unnatural: distractor is grammatical but no native would say
      it that way.
    - Length-symmetry: 4 choices should be roughly similar in
      character count for moji-style questions.

D6. Ko-so-a-do questions lacking scene context
    - これ vs それ vs あれ is determined by spatial/conversational
      proximity. Without scene-setting in the stem, the question is
      ambiguous. Reference: §3.2.4 + §C.2 in procedure manual.

D7. Placeholder distractor explanations
    - "See pattern detail" or "Refer to grammar.json" in
      rationale_en / rationale_hi — flag as MAJOR. Each distractor
      explanation must teach why the wrong answer is wrong.
    - Reference: §3.2.3.

D8. Filler-MCQ smell
    - When the question feels auto-generated rather than authored —
      uniform sentence templates across many questions, distractors
      that look picked at random from a vocab list, no testable
      learning point. Reference: §3.2.1.

Category E — PROVENANCE + CROSS-REFERENCE INTEGRITY (MINOR severity)
--------------------------------------------------------------------

E1. Cross-references to retired / renamed pattern IDs
    - The 2026-05 vocab + grammar dedup retired several IDs;
      check `_meta.retired_pattern_ids` and verify no example /
      common_mistake / wrong_corrected_pair references them.

E2. Vocab_ids that don't resolve
    - Example sentence carries vocab_ids that don't exist in
      vocab.json. Flag.

E3. Audio refs not in audio_manifest
    - listening.json or reading.json items with audio paths not
      listed in data/audio_manifest.json — fall through to "audio
      not yet generated" in the UI.

E4. meaning_ja / meaning_en / meaning_hi drift
    - Three locale fields for the same pattern saying semantically
      different things. One is current, others are stale. Flag any
      semantic divergence.

E5. Form-field consistency on examples
    - When some examples in a pattern have `form` set and others
      don't, the UI renders empty `_` badges (user-visible drift).
      Either all examples in a pattern have form, or none.
      Reference: §3.2.34.

E6. SCHEMA DRIFT — same field, different key names (added 2026-05-13)
    Audit for inconsistent schema across the same logical field:
      - Listening lines: `text_ja` (legacy) vs `ja` (canonical).
        Fixed 2026-05-13 (145 entries renamed); JA-canonical now.
      - Reading questions: `question_ja` (legacy) vs `prompt_ja`
        (canonical). Fixed 2026-05-13 (9 entries renamed);
        JA-73 locks forward.
      - Listening speaker tags: `M`/`F` (legacy single-letter) vs
        `male`/`female` (canonical). Fixed 2026-05-13 (13 normalized).
      - Vocab readings: slash-separated string `"なに / なん"` (legacy)
        vs `readings: ["なに", "なん"]` list (canonical). Fixed 2026-05-13;
        JA-74 locks forward.
    Test: for any field, count distinct key-name spellings used. If
    >1, flag as schema drift even if functionally equivalent.

E7. SEE_ALSO TARGET RESOLUTION (added 2026-05-13)
    When a `see_also` field is present, its values must resolve to
    actual pattern IDs or pattern-marker strings that exist
    elsewhere in grammar.json. Failure modes:
      - Empty placeholder target: `see_also: ["〜"]`
      - Abstract category: `see_also: ["Verb"]`, `["Adjective + Noun"]`
      - Slash-separated reading: `see_also: ["なん / なに"]`
    These were artifacts of the 2026-05-13 G-2 cleanup; verify any
    new see_also additions resolve.

E8. ABOVE-TARGET-LEVEL KANJI IN ENGLISH COMMENTARY FIELDS (added 2026-05-13)
    Anti-pattern #21 (procedure manual D.9.21). The N5 master kanji-
    scope check (JA-13) leaf-skips `explanation_en`, `pattern_role`,
    `gloss`, `translation_en`, `rationale` — these are nominally
    English commentary. BUT the corpus convention is that even
    inside English commentary, Japanese words referenced should use
    only target-level (N5) kanji + kana for above-level words.
    Failure mode example (caught Phase-7 polish 2026-05-13):
      explanation_en: "Examples: 大好き / 大嫌い / 広い / 読み方"
      → 好/嫌/広/方 are all N4+; should be in kana within the gloss.
    CI invariant JA-66 catches this. When sampling, scan
    `explanation_en` and `public_domain_refs.pattern_role` for
    above-N5 kanji manually.

WHAT IS OUT OF SCOPE — DO NOT FLAG
============================================================

INTENTIONAL CROSS-LEVEL REGISTER SURFACES
-----------------------------------------

These fields are DESIGNED to show N3+ register variation for
pedagogical context. They are explicitly exempted from JA-13
(out-of-scope-kanji invariant) via SKIP_SUBTREE_FIELDS in
tools/check_content_integrity.py. DO NOT generate findings flagging
N3+ vocab / kanji as out-of-scope when they appear in:

- `politeness_ladder.humble` (e.g. たこうございますね — N3 keigo)
- `politeness_ladder.respectful` (e.g. 〜られる尊敬語 forms)
- `wrong_corrected_pair` (intentionally shows native-grade wrong/right
  pairs; N4+ vocab in the "wrong" example is THE POINT — it
  illustrates a real error a learner would make)
- `common_mistakes` (Japanese illustration uses native vocab)
- `cultural_context` (background notes use whatever Japanese is
  needed for the cultural point)
- `summary` (pedagogical commentary uses normal Japanese)
- `authentic_citations` (textbook bibliographic metadata — chapter +
  context note format, uses normal Japanese for the meta-commentary)

DO flag actual Japanese errors (wrong conjugation, wrong particle,
half-applied ウ音便, etc.) inside these surfaces — they are exempted
from kanji-scope-only, not from Japanese-correctness.

LOCALE-SUFFIXED TRANSLATION FIELDS
----------------------------------

These hold L1 translations (Hindi, English) — never apply Japanese
grammar / orthography rules to them:
- meaning_hi, meaning_en
- explanation_hi, explanation_en
- gloss_hi, gloss_en
- summary_hi, summary_en
- title_hi, title_en
- prompt_hi, prompt_en
- description_hi, description_en
- note_hi, note_en
- meanings_hi, meanings_en
- Any field ending in _hi, _en, _vi, _id, _ne, _zh.

Hindi prose carrying Japanese grammatical particles must use kana
not romaji (R-1.1 invariant JA-41) — that IS in scope.

MOCK-PAPER FORMAT ADHERENCE — paper-distractor exception
--------------------------------------------------------

data/papers/moji/, data/papers/goi/, data/papers/bunpou/ use the
official JLPT exam-paper format where DISTRACTORS may use N4+ kanji
to test recognition. Flag only if the CORRECT ANSWER uses out-of-N5
kanji. Reference: JA-40 invariant.

NORMALIZATION CASES THAT ARE INTENTIONAL
----------------------------------------

- いい / よい — both forms are correct N5; documented inconsistency
  by register (いい casual, よい formal).
- Some patterns use 漢字 form in stems and 仮名 form in choices —
  this is testing kanji-reading, not orthographic inconsistency.

KNOWN CHECKER FALSE POSITIVES (do NOT re-flag these — they ARE

These were caught and confirmed-as-not-bugs in earlier audit passes.

FP-1. ウ音便 contraction where o+u → ō has no visible spelling change
      (added 2026-05-14)

      Examples of CORRECT forms that some checkers flag as
      "half-applied ウ音便":
        おもしろう ございます  ← おもしろい (omoshiroi) → onbin
                                  drops く: omoshirou → o+u → ō
                                  Written: stays as おもしろう because
                                  the long ō is spelled with う in
                                  modern Japanese (cf. ありがとう).
        あおう ございます      ← あおい (aoi)   → o+u → ō
        あかう ございます      ← あかい (akai)  → a+u → ō (written
                                  あこう in some styles; both accepted)

      Compare CORRECT contractions where spelling DOES change:
        おいしゅう ございます  ← おいしい — i+u → yū (spelling changes)
        たこう ございます      ← たかい  — a+u → ō (spelling changes:
                                  たかう → たこう)

      RULE OF THUMB: vowel before う in the onbin form:
        - "a" → spelling changes to "o" (たかう → たこう)
        - "i" → spelling changes to "yū" (おいしう → おいしゅう)
        - "o" → spelling stays "ou" (おもしろう, あおう)
        - "u" → spelling stays "uu" (やすう)

      If the auditor sees a form already-ending-in-"ou"+ございます,
      check whether the i-adjective stem ended in "o" before flagging.

FP-2. Past-marker mid-sentence in relative clauses
      (added 2026-05-14)

      Example: わたしが 買った 本 (the book I bought)
      Form tag: "past"
      Naive end-of-sentence check: NO past marker found (sentence
        ends in "本" which is a noun).
      Reality: 買った IS the past form, just inside a relative clause
        modifying 本. The form-tag is correct.

      RULE: a sentence with a relative-clause modifier on the LAST
      noun phrase doesn't end with the past marker, but is still
      past-tense overall. Don't flag.

FP-3. Mixed kanji/kana orthography within one pattern
      (added 2026-05-14)

      Example: pattern n5-007 で has 学校 (kanji) in [2] and がっこう
        (kana) in [7]. Both are accurate Japanese; the project has
        no enforced single-spelling convention.

      This IS a real style/consistency issue (see §3.2.17 in the
      procedure manual), but it is NOT a language-ACCURACY bug.
      Surface as a style note, NOT a CRITICAL finding.

FP-4. Cross-pattern reuse of canonical sentences below threshold-10
      (added 2026-05-14)

      Example: "わたしは がくせいです。" appears in 9 patterns. All
      9 are basic copula / identity patterns where the sentence IS
      a legitimate canonical example. The CI gate JA-81 catches
      sentences at frequency 10+; below that, manual judgment
      applies.

      RULE: only flag cross-pattern reuse when EITHER:
        (a) frequency ≥ 10 (JA-81 would fail);
        (b) frequency ≥ 4 AND ≥2 of the occurrences are in patterns
            whose marker is NOT visible in the sentence (genuine
            misplaced boilerplate, not canonical reuse).

FP-5. Counter rendaku in headword-presence checks
      (added 2026-05-15)

      Counter vocab entries headed ひき / はい / ふん / ほん / etc.
      legitimately appear in examples as ぴき / ばい / ぷん / ぽん
      (rendaku / handakuon alternation triggered by preceding number):
        - n5.vocab.9-counters-common.ひき [2] = 'うちには ねこが 三びき います。'
        - n5.vocab.9-counters-common.はい [0] = 'コーヒーを 一ぱい 飲みました。'
      These are CORRECT examples. A naive substring check for "ひき"
      in the ja text will miss "びき" and false-flag. Apply the
      rendaku alternant set (see A23b) before flagging.

FP-6. Compound -くる verbs in headword-presence checks
      (added 2026-05-15)

      Compound verbs ending in くる (e.g. もってくる, つれてくる)
      conjugate on the くる auxiliary:
        - もってくる → te-form もってきて, past もってきました
      The kana "くる" never appears in conjugated examples. A naive
      substring check misses these. Expand the stem set to
      prefix + {き, こ, く} for any reading containing くる.

FP-7. Demonstrative-pronoun entries using 'これは Xです。'
      (added 2026-05-15)

      For these entries: これ, それ, あれ, この, その, あの, here-style
      and there-style demonstratives in section 5-demonstratives.
      Examples like 'これは ほんです。' / "This is a book." are the
      CANONICAL demonstration of the headword. They are not
      boilerplate — the headword is the demonstrative itself, and
      the template IS the lesson.

      RULE for A20 (KORE-BARE template): skip when entry.pos =
      "demonstrative" OR section contains "demonstrative". The
      A20 detector hard-codes this exception.

FP-8. English translations with trailing parenthetical annotations
      (added 2026-05-15)

      Vocab examples often carry pedagogical annotations after the
      sentence terminator:
        - "I am a student. (topic は)"
        - "Good morning. (formal)"
        - "It is my book. (possessive の)"
      The sentence itself IS terminated; the annotation is metadata.
      A naive "ends with .?!" check sees the closing ")" and false-
      flags. Strip trailing /\s*\([^)]*\)\s*$/ BEFORE checking for
      terminator.

FP-9. authentic.json `reading` field is the FULL phrase read-aloud
      (added 2026-05-15 wave-1)

      The authentic.json schema's `reading` field captures the
      verbalised pronunciation of the WHOLE ja phrase, not the
      single-vocab-entry reading. For card auth.menu.koohii:
        ja:      'コーヒー 350円'           (the signage as written)
        reading: 'コーヒー さんびゃくごじゅうえん'  (how a Japanese speaker
                                            would read the sign aloud)
        vocab_refs: ['n5.vocab.18-drinks.コーヒー']  (which vocab entries
                                                    appear inside)
      A cross-corpus reading-vs-vocab.json check that doesn't condition
      on "ja == canonical form" will false-flag 16 such multi-word
      signage cards. Apply the check ONLY when ja matches a single
      vocab form exactly.

FP-10. English/short-technical rationales for paper-question answers
       (added 2026-05-15 wave-3)

       Paper-question rationales are often brief English paraphrases
       or grammar-tag micro-explanations that legitimately don't share
       Japanese characters with the (Japanese) correct answer:
         answer '来ました'      rationale 'past affirmative (came one
                                          month ago).'
         answer '220えん'       rationale '1000 - (100×3) - (120×4) =
                                          1000 - 300 - 480 = 220.'
         answer 'まいにち'       rationale '毎 (マイ) + 日 (ニチ).'
       A char-overlap heuristic between correct-answer and rationale
       flags 115 of 402 paper-questions as misaligned, but manual
       triage finds only ~2% are real misalignments. Flag only when
       the rationale CLEARLY describes a different passage segment
       than the answer (see A30).

FP-11. ひとりでに ("by oneself") is a fixed adverb, not a particle stack
       (added 2026-05-15 wave-2)

       The substring 「でに」 appears inside the compound adverb
       「ひとりでに」 (= "by oneself / spontaneously", e.g. 「まどが
       ひとりでに あきました」 = "the window opened by itself").
       This is a single lexicalised word, not 「で」 + 「に」 stacked
       as separate particles. A regex catching「でに」 as an illegal
       particle stack must use a lookbehind `(?<!ひとり)` to exclude
       this fixed expression.

FP-12. 〜いです IS the correct polite-affirmative form of an i-adj
       (added 2026-05-15 native-teacher pass)

       The form `<i-adj-stem>いです` (e.g. 「おもしろいです」,
       「おいしいです」, 「あついです」) is the CANONICAL polite-
       affirmative form of an i-adjective. It is NOT the "should-be
       くて" wrong-conjunction form. The genuinely wrong forms are:
         <stem>いで、 (should be <stem>くて、)
         <stem>いでは  (should be <stem>くては)
         <stem>いでも  (should be <stem>くても)
       A regex catching the wrong forms MUST exclude `で(?=す)` from
       the trigger set, or it will false-flag 240+ correct polite-
       affirmative forms across the corpus.

FP-13. 倍 (ばい) is a multiplier suffix, not a noun-counter
       (added 2026-05-15 native-teacher pass)

       三倍 (sanbai = "triple"), 二倍 (nibai = "double") attaches to
       numbers as a multiplier suffix. It is NOT a noun-counter (no
       「Xを 三倍」 with a real noun). A counter-noun-pairing audit
       (A37) must EXCLUDE 倍 from the counter token set. Otherwise
       「三ばいに なりました」 (became triple) will be flagged as a
       noun + counter pair.

FP-14. i-adjective inflected forms still demonstrate the headword
       (added 2026-05-15 native-teacher pass)

       i-adjectives inflect by dropping the final い:
         dictionary:    おもしろい
         te-form:       おもしろくて
         negative:      おもしろくない
         past:          おもしろかった
         adverbial:     おもしろく
       Any of these inflected forms CONTAINS the i-adj headword
       semantically — the entry IS being demonstrated. A naive
       substring search for the full dictionary form 「おもしろい」
       will miss 「おもしろく / おもしろくて / おもしろかった」 and
       false-flag the example as "headword missing". The check must
       also accept stem-only (drop final い) as a valid match.

       Same caveat for verb-3 'する' standalone (stems are 'し' and
       'す', not just 'する'): 「しゅくだいを します」 contains 'し'
       (in します), so the headword IS present.

FP-15. ぜひ / ただ + neutral declarative — semantically thin, not a
       grammar bug
       (added 2026-05-15 native-teacher pass)

       Adverbs ぜひ and ただ have natural pairings:
         ぜひ + volitional / request: ぜひ 来てください, ぜひ 食べたい
         ただ + minimization:         ただの 友だち, ただ 見るだけ
       A neutral-declarative pairing like 「ぜひ ごはんを 食べます」
       is grammatically valid but semantically thin — the adverb
       adds nothing meaningful. This is a pedagogical-quality issue
       (caught at vocab.adverb entries' example[1] slot), not a
       grammar error. Treat as "should improve" not "must fix".
