import re
from collections import Counter
pat_miru  = re.compile(r"^[^\s]{1,10}гӮ’\s*иҰӢгҒҫгҒ—гҒҹ(?:гӮҲ|гҒӯ|гӮҲгҒӯ)?гҖӮ?$")
pat_doko  = re.compile(r"^гҒӮгҒ®\s+(.{1,10})гҒҜ\s*гҒ©гҒ“гҒ§гҒҷгҒӢгҖӮ?$")
pat_kore  = re.compile(r"^(гҒ“гӮҢ|гҒӮгӮҢ|гҒқгӮҢ)гҒҜ\s*(.{1,10})гҒ§гҒҷгҖӮ?$")
pat_quote = re.compile(r"^гҖҢ(.{1,10})гҖҚгҒЁ\s*гҒӮгҒ„гҒ•гҒӨгҒ—гҒҫгҒ—гҒҹгҖӮ?$")
GREETINGS = {"гҒҠгҒҜгӮҲгҒҶ","гҒ“гӮ“гҒ«гҒЎгҒҜ","гҒ“гӮ“гҒ°гӮ“гҒҜ","гҒӮгӮҠгҒҢгҒЁгҒҶ","гҒ•гӮҲгҒҶгҒӘгӮү",
             "гҒҠгӮ„гҒҷгҒҝ","гҒҹгҒ гҒ„гҒҫ","гҒҠгҒӢгҒҲгӮҠ","гӮӮгҒ—гӮӮгҒ—","гҒ„гҒЈгҒҰгҒҚгҒҫгҒҷ",
             "гҒ„гҒЈгҒҰгӮүгҒЈгҒ—гӮғгҒ„","гҒҠгҒӢгҒ’гҒ•гҒҫгҒ§"}
n_miru = n_doko = n_kore = n_greet = n_dup = 0
ja_xrefs = Counter()
for v in V:
    section = v.get('section','')
    pos = v.get('pos','')
    form = v.get('form','')
    is_loc = '13-locations' in section or '26-house' in section
    is_dem = pos == 'demonstrative' or 'demonstrative' in section.lower()
    exs = v.get('examples', [])
    seen = {}
    for i, ex in enumerate(exs):
        ja = (ex.get('ja') or '').strip()
        if pat_miru.fullmatch(ja): n_miru += 1
        m = pat_doko.fullmatch(ja)
        if m and not is_loc and m.group(1).strip() == form.strip(): n_doko += 1
        m = pat_kore.fullmatch(ja)
        if m and not is_dem and len(exs) >= 3: n_kore += 1
        m = pat_quote.fullmatch(ja)
        if m and m.group(1) not in GREETINGS: n_greet += 1
        if ja and ja in seen: n_dup += 1
        elif ja: seen[ja] = i
        ja_xrefs[ja] += 1
print('Template XгӮ’ иҰӢгҒҫгҒ—гҒҹ leak (A18, target 0):', n_miru)
print('Template гҒӮгҒ® XгҒҜ гҒ©гҒ“ non-location (A19, target 0):', n_doko)
print('Template гҒ“гӮҢгҒҜ XгҒ§гҒҷ bare on 3+ex non-demonstrative (A20, target 0):', n_kore)
print('Template гҖҢXгҖҚгҒЁ гҒӮгҒ„гҒ•гҒӨгҒ—гҒҫгҒ—гҒҹ non-greeting (A21, target 0):', n_greet)
print('Within-entry duplicate ja (A17/vocab, target 0):', n_dup)
print('Cross-entry boilerplate вүҘ5x (A22, target 0):',
      sum(1 for c in ja_xrefs.values() if c >= 5))

    re.compile(r'(?:гҒ§гҒҷ|гҒҫгҒҷ)гҒӢгҒӢ'),

    return sum(1 for rx in PATTERNS if rx.search(text or ''))

## LESSONS FROM PRIOR CYCLES (don't relitigate these)

These lessons came from 13 audit cycles producing ~185 content fixes
across 2026-04 through 2026-05-13. The audit MUST respect them; doing
so prevents loops where each new cycle re-discovers the same gap.

### Anti-pattern A: ad-hoc sampling at saturation produces false positives
Run-3 of 2026-05-13 (under the prior version of THIS prompt) "found"
several issues that did not exist on inspection вҖ” they came from
inspecting 6 random items and pattern-matching against memory of run-1
findings. The fix: Phase-0 mechanical checklist (Step 1.6 above)
produces the actual numbers; qualitative findings are layered on top
of mechanical numbers, never instead of them.

### Anti-pattern B: human-day effort estimates are invalid
The audit MUST NOT estimate effort in human-person-days. The audit
runs in Claude, the fixes are applied by Claude, the reviewer is one
human. Use Claude-turns: "1 turn", "2-3 turns", "5+ turns". A turn
is a single Claude response cycle. This was added to CLAUDE.md as
anti-pattern #22.

### Anti-pattern C: calendar-cadence audit reruns produce zero signal at saturation
The DO-NOT-RUN-ON-CALENDAR block at line 1 of this file exists
because three runs in one working session produced 29 / 5 / 0
actionable findings. The third run was confirmatory only and added
zero value. Calendar reruns (every Friday, every release, every PR)
spend reviewer attention on confirmatory output. Run ONLY on trigger
conditions (see line 1).

### Anti-pattern D: scorecard cells calcify against the live corpus
The 2026-05-13 cycle found 9 wrong field-name lookups producing
false-0% readings. The cells in this prompt represent what the
authors EXPECTED the field-names to be; the live data is the source
of truth. Step 1.5 (Field-Name Verification) fires automatically
before any "below bar" claim.

### Anti-pattern E: width-additions are out of scope for richness audits
Every cycle, the temptation is "let's add 20 more vocab to fill
gaps." The RICHNESS-FIRST mandate (line ~200) freezes widths.
Depth on EXISTING entries is the work. If the audit produces "add
N more vocab" as a P1 finding, the prompt has lost the richness-
first discipline and should NOT be acted on without explicit
maintainer override.

### Anti-pattern F: provenance tier is not a user-facing trust signal
The `native_reviewed` provenance tag is a TRACKING LABEL ONLY вҖ”
nothing in the corpus has been reviewed by a Japanese native speaker;
the maintainer hand-curated and LLM-promoted entries carry this tag.
Exposing the tag in UI as a trust signal would be a false claim.
IMP-184 was set to Avoid 2026-05-13 for this reason. Do not surface
provenance to learners; the content's correctness is the trust signal.

## INTERCONNECTION-DENSITY CHECK (a first-class richness dimension)

Content richness = per-entry depth Г— cross-reference density. The
audit must produce these numbers:

```
Density-1: every grammar pattern вҶ’ how many vocab entries link IN via vocab_ids? (target: вүҘ10 average; вүҘ3 floor per pattern)
Density-2: every vocab entry вҶ’ how many grammar patterns reference it? (target: вүҘ2 average; вүҘ1 floor per word)
Density-2b (REVERSE): every vocab entry вҶ’ which patterns it FREQUENTLY appears with (e.g., йЈҹгҒ№гӮӢ вҶ’ ~гҒҹгҒ„, ~гҒҹгҒ“гҒЁгҒҢгҒӮгӮӢ, ~гҒӘгҒҢгӮү). This is a different field from "patterns that reference this word" вҖ” Density-2 is grammarвҶ’vocab; 2b is vocabвҶ’pattern co-occurrence. (target: вүҘ3 patterns per high-frequency content word)
Density-3: every kanji вҶ’ how many vocab entries use this kanji? (target: вүҘ5 average; вүҘ2 floor per kanji)
Density-4: every kanji вҶ’ how many reading passages contain this kanji? (target: вүҘ3 average; floor вүҘ1 per kanji)
Density-5: every reading passage вҶ’ how many sentences have inline grammar-footnote links? (target: 100% of sentences)
Density-6: every listening item вҶ’ vocab in transcript clickable to vocab page? (target: 100%)
Density-7: every paper question вҶ’ does it cross-link to its grammar/vocab/kanji entry on review? (target: 100%)
Density-8: every confusable kanji cluster вҶ’ all members linked to all others? (target: 100%)
Density-9 (NEW): every grammar pattern with register variants вҶ’ does it cross-link to the politeness ladder (casual/polite/humble/respectful chain)? (target: 100%)
Density-10 (NEW): every vocab entry with pragmatic-multi-function вҶ’ does it enumerate all functions inline? (target: 100% of multi-function entries)

Aggregate metric: total cross-references / sqrt(corpus size)
This number is the single most underrated quality metric.
A learner-facing app where every entry is a dead-end is "shallow"
even if each entry is itself well-authored.
```

The audit must report each density number plus the aggregate. The
priority of fixes follows from which density has the worst gap.

### Particle-precision audit (гҒҜ vs гҒҢ is the #1 error source)

For every grammar example and question stem using гҒҜ or гҒҢ,

### Verb-class & inflection trap audit

- Group-1 exception verbs (X-6.6): е…ҘгӮӢ/её°гӮӢ/иө°гӮӢ/зҹҘгӮӢ/еҲҮгӮӢ/иҰҒгӮӢ +
  Bunpro lists е–ӢгӮӢ but it's late-N5+. Each must carry the
  `group1_exception: true` flag AND have its negative + te + ta
  forms tested in at least one item to verify the exception is
  honored, not just labelled.
- Irregular te-form: иЎҢгҒҸ вҶ’ иЎҢгҒЈгҒҰ (NOT иЎҢгҒ„гҒҰ). Audit must verify
  every иЎҢгҒҸ-related example uses the correct te-form.
- гҒҷгӮӢ-compound verbs vs lexical гҒҷгӮӢ: еӢүеј·гҒҷгӮӢ is composed
  (гҒ№гӮ“гҒҚгӮҮгҒҶ + гҒҷгӮӢ); й–ўгҒҷгӮӢ is lexical. Different conjugation
  patterns; audit per-entry.
- жқҘгӮӢ reading shifts: жқҘгӮӢ (гҒҸгӮӢ), жқҘгҒҫгҒҷ (гҒҚгҒҫгҒҷ), жқҘгҒҹ (гҒҚгҒҹ),
  жқҘгҒҰ (гҒҚгҒҰ), жқҘгҒӘгҒ„ (гҒ“гҒӘгҒ„). The kun reading shifts by form;
  audit confirms each form's reading is correctly transcribed.

- гҒ©гҒҚгҒ©гҒҚ (heart pounding), гӮҸгҒҸгӮҸгҒҸ (excited),
- гҒҙгҒӢгҒҙгҒӢ (sparkling/clean), гӮҶгҒЈгҒҸгӮҠ (slowly),
- гҒЎгӮҮгҒЈгҒЁ (a little вҖ” pragmatic), гҒ гӮ“гҒ гӮ“ (gradually),

### Politeness ladder explicit teaching

For each grammar pattern with register variants, audit verifies
the ladder is shown EXPLICITLY (not just "casual / polite" tag):
- Causation ladder: гҒӢгӮү вҶ’ гҒ®гҒ§ вҶ’ гҒӘгҒ®гҒ§ вҶ’ гҒ§гҒҷгҒ®гҒ§ вҶ’ гҒ§гҒ”гҒ–гҒ„гҒҫгҒҷгҒ®гҒ§
- Existence ladder: гҒ„гӮӢ вҶ’ гҒ„гӮүгҒЈгҒ—гӮғгӮӢ вҶ’ гҒҠгӮӢ
- Eating ladder: йЈҹгҒ№гӮӢ вҶ’ еҸ¬гҒ—дёҠгҒҢгӮӢ вҶ’ гҒ„гҒҹгҒ гҒҸ
- Saying ladder: иЁҖгҒҶ вҶ’ гҒҠгҒЈгҒ—гӮғгӮӢ вҶ’ з”ігҒҷ
- Going/coming ladder: иЎҢгҒҸгғ»жқҘгӮӢ вҶ’ гҒ„гӮүгҒЈгҒ—гӮғгӮӢ вҶ’ еҸӮгӮӢ

ladder.

### Pragma-multi-function annotation (single-gloss is wrong)
