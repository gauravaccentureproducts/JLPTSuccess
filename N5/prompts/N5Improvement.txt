  - **Migaku** — sentence-mining from authentic media, Chrome

    voice talent, cultural notes per lesson.
  - **Anki + shared decks** — free SRS.
  - **JLPT Sensei** — free web reference; thin per-pattern

## INTERCONNECTION-DENSITY CHECK (a first-class richness dimension)

Content richness = per-entry depth × cross-reference density. The
audit must produce these numbers:

```
Density-1: every grammar pattern → how many vocab entries link IN via vocab_ids? (target: ≥10 average; ≥3 floor per pattern)
Density-2: every vocab entry → how many grammar patterns reference it? (target: ≥2 average; ≥1 floor per word)
Density-2b (REVERSE): every vocab entry → which patterns it FREQUENTLY appears with (e.g., 食べる → ~たい, ~たことがある, ~ながら). This is a different field from "patterns that reference this word" — Density-2 is grammar→vocab; 2b is vocab→pattern co-occurrence. (target: ≥3 patterns per high-frequency content word)
Density-3: every kanji → how many vocab entries use this kanji? (target: ≥5 average; ≥2 floor per kanji)
Density-4: every kanji → how many reading passages contain this kanji? (target: ≥3 average; floor ≥1 per kanji)
Density-5: every reading passage → how many sentences have inline grammar-footnote links? (target: 100% of sentences)
Density-6: every listening item → vocab in transcript clickable to vocab page? (target: 100%)
Density-7: every paper question → does it cross-link to its grammar/vocab/kanji entry on review? (target: 100%)
Density-8: every confusable kanji cluster → all members linked to all others? (target: 100%)
Density-9 (NEW): every grammar pattern with register variants → does it cross-link to the politeness ladder (casual/polite/humble/respectful chain)? (target: 100%)
Density-10 (NEW): every vocab entry with pragmatic-multi-function → does it enumerate all functions inline? (target: 100% of multi-function entries)

Aggregate metric: total cross-references / sqrt(corpus size)
This number is the single most underrated quality metric.
A learner-facing app where every entry is a dead-end is "shallow"
even if each entry is itself well-authored.
```

The audit must report each density number plus the aggregate. The
priority of fixes follows from which density has the worst gap.

  patterns — "this exact pattern appears in [show] episode N at

- **News headline examples** for late-N5 patterns where N5 grammar

## PEDAGOGICAL CORRECTNESS (native-teacher dimensions)

These dimensions are what an experienced N5 classroom teacher
would catch but a UI-focused audit wouldn't. Each is a distinct
audit area; each item below maps to its own scorecard row above
or to a Section-5 / Section-6 finding.

### Particle-precision audit (は vs が is the #1 error source)

For every grammar example and question stem using は or が,
audit must verify:
- Information-structure annotation: is the topic GIVEN
  (は-marker) or NEW (が-marker)?
- Predicate type compatibility (state predicates → が-bias for
  experiencer; action predicates → は-bias for topic)
- "Exhaustive listing" reading of が is flagged where applicable
- 〜は〜が double-particle constructions explicitly explained

### Verb-class & inflection trap audit

- Group-1 exception verbs (X-6.6): 入る/帰る/走る/知る/切る/要る +
  Bunpro lists 喋る but it's late-N5+. Each must carry the
  `group1_exception: true` flag AND have its negative + te + ta
  forms tested in at least one item to verify the exception is
  honored, not just labelled.
- Irregular te-form: 行く → 行って (NOT 行いて). Audit must verify
  every 行く-related example uses the correct te-form.
- する-compound verbs vs lexical する: 勉強する is composed
  (べんきょう + する); 関する is lexical. Different conjugation
  patterns; audit per-entry.
- 来る reading shifts: 来る (くる), 来ます (きます), 来た (きた),
  来て (きて), 来ない (こない). The kun reading shifts by form;
  audit confirms each form's reading is correctly transcribed.

### Onomatopoeia / mimetic words (擬音語/擬態語)

Even at N5, real Japanese is mimetic-heavy. Audit must verify
the corpus includes at least these N5-level mimetics with full
schema (form, gloss, gloss_hi, examples, audio):
- ぺこぺこ (very hungry), にこにこ (smiling),
- どきどき (heart pounding), わくわく (excited),
- ぴかぴか (sparkling/clean), ゆっくり (slowly),
- ちょっと (a little — pragmatic), だんだん (gradually),
- もっと (more), まあまあ (so-so).

If the corpus has 0/10 of these, that's the largest leverage
gap on the vocab surface.

### Politeness ladder explicit teaching

For each grammar pattern with register variants, audit verifies
the ladder is shown EXPLICITLY (not just "casual / polite" tag):
- Causation ladder: から → ので → なので → ですので → でございますので
- Existence ladder: いる → いらっしゃる → おる
- Eating ladder: 食べる → 召し上がる → いただく
- Saying ladder: 言う → おっしゃる → 申す
- Going/coming ladder: 行く・来る → いらっしゃる → 参る

Native teachers explicitly show the ladder; textbooks underplay
it. Audit flags any pattern with register variants but no
ladder.

### Pragma-multi-function annotation (single-gloss is wrong)

Audit verifies every entry below has its multiple speech-act
functions glossed inline:
- すみません = {apology, attention-getter, gratitude-when-receiving}
- 大丈夫 = {confirmation, polite-refusal — "I'm OK, no thanks"}
- どうぞ = {offering, please-do, by-all-means}
- どうも = {thanks-shortened, sorry-shortened, hello-shortened, very}
- ちょっと = {a-little, pragmatic-softener "ちょっと…"-decline}
- 結構 = {fine, no-thanks-polite-refusal}

Single-translation glosses miss the function. Audit flags any
of these entries that doesn't enumerate functions.

### Phonological discrimination (listening / pitch / vowels)

- **Pitch minimal pairs**: 雨/飴, 橋/箸, 神/紙, 牡蠣/柿, 雲/蜘蛛.
  At N5 level, audit must verify ≥3 of these are present with
  pitch annotation + audio of both readings.
- **Long-vowel discrimination**: ビル/ビール, おばさん/おばあさん,
  おじさん/おじいさん, ここ/こうこう (school).
- **Sokuon discrimination**: 来た/切った, 学生/学期, きて/きって.
- **Devoiced vowels**: です ("des'"), すき ("sk'"), きく ("kk"),
  した ("sht'"). Audit confirms transcripts mark the devoicing.

### Spacing convention (分かち書き) graduation

- Early N5 content uses 分かち書き (spaces between words):
  「私は 学生です」.
- Late-N5 / N4-bridge content MUST graduate to unspaced:
  「私は学生です」.

Audit flags: corpus has spacing applied uniformly; learners
never see real Japanese spacing convention; graduation strategy
is missing.

### Furigana display strategy

- Each route persists a furigana mode independently:
  always-show / hide-when-known / first-occurrence-only / never.
- Audit verifies the mode persists per route (not just globally)
  and is reset-able.
- Default for early-N5 routes: always-show; default for N4-bridge
  routes: hide-when-known.

### Romaji discipline

A serious JLPT app shows ZERO romaji to learners (defeats kana
literacy). Audit verifies:
- No romaji in user-facing surfaces (vocab, grammar, kanji,
  reading, listening pages).
- Romaji INPUT in typed-answer mode is acceptable (matches
  Bunpro's IME-assist).
- Any romaji in tool/debug code is fine; audit only flags
  user-visible romaji.

### Hand-writing pedagogy gap

- Native kanji learning is hand-writing-first (book reading is
  recognition-only).
- Audit acknowledges: current app is recognition-only.
- Audit decision: file as IMP (improvement) with HIGH effort
  (would require canvas-based stroke capture + scoring).
- If decision is "do not ship," log as Section 10 anti-item with
  rationale (out of scope for static PWA).

### Cultural-context callout enumeration (canonical 12 for N5)

Audit must verify each appears at least once in the corpus
(reading passages or grammar/vocab callouts):
1. School calendar — 4-month start, 3-month end
2. Education tier — 小→中→高→大
3. Major holidays — お正月 (Jan 1), GW (early May), 七夕 (Jul 7),
   お盆 (mid-Aug), クリスマス (commercial only)
4. Train system — JR / 私鉄 / 地下鉄
5. Workplace — 年功序列, 正社員 vs アルバイト
6. Polite-language reflexes — 頑張ります, よろしくお願いします,
   お疲れさまです
7. Greeting drift — おはようございます = MORNING ONLY
8. Meal rituals — いただきます / ごちそうさま
9. Room-entering ritual — 失礼します
10. Dietary defaults — 米 staple, お茶 default drink
11. Currency — 円, no cents/decimals
12. Address order — 大→小: 都道府県→市→区→町→番地

## TEST-TAKING STRATEGY (受験対策) — JLPT-specific layer

Real JLPT prep books include a strategy layer that no incumbent
app teaches systematically. This is a high-leverage richness
dimension. Audit must verify the corpus / UI surfaces include:

### Per-mondai timing strategy

Test-mode timer shows an internal pacing recommendation per
mondai, not just the section total:
- 言語知識（文字・語彙）25min: M1 漢字読み (5min) / M2 表記 (5min) /
  M3 文脈規定 (10min) / M4 言い換え類義 (5min)
- 言語知識（文法）・読解 50min: M1 文の文法1 (15min) / M2
  文の文法2 (5min) / M3 文章の文法 (5min) / M4-6 読解 (25min)
- 聴解 30min: fixed pace by audio (no per-mondai discretion)

### Trap-pattern catalog (引っかけパターン)

Audit verifies the corpus / explanation surfaces document the
common JLPT N5 traps:
- 似た意味 (similar meaning) distractors in 言い換え類義 — MUST
  ship at least 5 worked examples with explicit "this looks
  right because... but is wrong because..." analysis
- 紛らわしい数字 (confusing numbers) in 聴解 — 七時 vs 一時 in
  fast speech, 4日 (yokka) vs 8日 (youka) discrimination
- 否定+否定=肯定 traps — N5-level double-negatives like
  「行かなくちゃいけない」 (must go) misread as "don't have to go"
- 順番引っかけ (sequence trick) in 課題理解 — speaker mentions
  multiple actions, only the last is the answer
- 漢字読み trap — same kanji compound has multiple readings;
  context disambiguates (一日 = いちにち / ついたち)

### Test-taking technique surfaces

- 消去法 (elimination): explicit "rule out two clearly wrong,
  pick between the remaining two" walk-through on a sample
  question per mondai type.
- 自信を持って先に進む (move on confidently): per-question
  budget reminder; flag-and-return-later UX.
- マークシート練習 (answer-sheet drill): mock-test mode shows
  the question number explicitly to mimic the bubble-sheet
  rhythm.
- 試験前夜 / 当日 routine note: pre-exam day checklist (sleep
  schedule, breakfast, what to bring) — short cultural artifact
  matching what Japanese cram schools provide.

### Score-breakdown diagnostic

End-of-paper results MUST surface:
- Per-mondai correct/total (NOT just per-section)
- Weighted score per section (lang+reading combined / listening)
- Total / 180 and pass/fail per the TWO post-2010 minimums
- Weakest mondai-type with one-click drill into that mondai
- Pace analysis: time spent per mondai vs recommended

If any of these are missing, file as Section-5 issue with
N3 niche-fit (test-mode realism is a defensible niche).

are 10 more." Audit verifies:

## PRIORITIZATION
