# JLPTSuccess プロジェクト概要（日本語版）

> **対象読者：** 日本語教育機関の教師、日本語教育学会員、JLPT
> 出題関係者、文部科学省日本語教育推進担当者など、本アプリを
> 教育的観点から評価しうる日本語母語話者。
> **目的：** 開発元から見て本アプリが何を志向し、何を志向していないか
> を一読で把握できるよう要約しました。詳細レビューは
> [`REVIEWER-PACK.ja.md`](REVIEWER-PACK.ja.md) を参照してください。
> **作成：** 2026-05-07（ラウンド9 終了時、v1.12.50）

---

## 1. 一文要約

**JLPTSuccess** は、登録不要・追跡なし・完全オフライン動作の
**JLPT N5 学習用ウェブアプリ** です。インド在住で日本語を
独学する学習者を主たる想定利用者とし、**ヒンディー語による
学習支援** を備えていることが他アプリとの主たる差別化点です。

## 2. プロジェクトの位置づけ

### 2.1 解こうとしている問題

日本語能力試験（JLPT）受験者数において、**インドは世界第5位**
（年間 約5万人受験）で、そのうち **約73% が N5 または N4 レベル**
です。一方、ヒンディー語を第一言語とする学習者向けの専用学習
アプリは事実上存在せず、既存の JLPT 対策アプリ（Bunpro,
WaniKani, Renshuu, Duolingo 等）はすべて英語または日本語ベース
です。

本プロジェクトはこの空白に対する一つの応答です。

### 2.2 想定利用者

- **インドの Tier-2 / Tier-3 都市の独学者：** 日本語学校・大学
  に通えず、月額有料サブスクリプションも利用できない学習者。
- **特定技能（SSW）ビザ志望者：** 受験まで6か月程度の期限
  プレッシャーがある社会人。
- **日本語を高校・大学で学ぶ学生：** 教科書の補助教材として。
- **インド国内の日本語教育機関：** 学校内ネットワークでの
  自己ホスティング先候補。

### 2.3 想定 *しない* 利用者

- **N4 以上の学習者：** N5 専用アプリです。N4 以降は別プロダクト
  予定（現在未着手）。
- **日本国内の日本語学習者：** 日本国内には既に十分な教材・教師
  資源があります。本アプリは「日本国外で独学する学習者」向けに
  最適化されています。
- **児童・幼児：** 大人の独学者を想定。ゲーミフィケーション
  （ストリーク・XP・ランキング）は意図的に排除しています。

## 3. 設計上の重要な選択

以下は本アプリの「アンチ・特徴」とも言える選択です。レビュー時
に「これが無いのは欠陥か？」とお考えになる前にご一読ください。

### 3.1 アカウント・ログイン無し

- すべての学習進捗は端末内 `localStorage` に保存。
- 端末をまたぐ同期機能は意図的に未実装。
- 復元には Settings 画面の「進捗エクスポート／インポート」
  （JSON ファイル）を使用。
- **理由：** 個人情報・メールアドレス収集を一切行わない設計。
  学校・NGO 等の機関導入時に GDPR 等のオーバーヘッドが
  発生しないことが「教育機関にとっての導入容易性」を担保します。

### 3.2 通信・トラッキング無し

- 初回読み込み後はオフラインで完全動作（PWA）。
- Google Analytics 等の解析ツール未搭載。
- 第三者スクリプト未搭載（CSP で同一オリジンのみ許可）。
- **理由：** インドの一部地域における通信量制限・
  データプライバシー懸念への配慮。「データはあなたの端末から
  出ない」を本アプリの最も差別化された主張と位置づけています。

### 3.3 広告・課金・有料サブスクリプション無し

- 個人利用は永続無料。
- 広告枠・有料機能・段階的有料化（freemium）はいずれも
  ロードマップに無し。
- 将来想定される収益源：教育機関向けの自己ホスティング支援
  契約（個人利用は引き続き無料）。
- **理由：** 「無料でアクセスできる教育的整合性のある JLPT N5
  教材」という公共財的位置づけを維持するため。

### 3.4 ゲーミフィケーション最小化

- ストリーク機能はあるが「あおる」UI は無し（「連続記録を
  途切らせるな！」のような通知は存在しない）。
- 経験値・レベルアップ・バッジは無い。
- **理由：** 大人の独学者向け。Duolingo 的な感情誘導は受験者
  にとってノイズです。

### 3.5 オリジナルコンテンツ

- 全コンテンツ（文法 178パターン、語彙 1,041 語、漢字 106 字、
  読解 45 文、聴解 47 問、模擬試験 29 紙 426 問）は **本プロ
  ジェクトによる独自著作** です。
- JLPT 過去問からの転載・引用・派生はありません。
- 自動著作監査（`tools/audit_provenance.py`）が「過去問署名」
  （年号付き出題マーカー、過去問用語、JEES 引用等）を毎リリース
  時にスキャンし、合致があればビルドを失敗させます。
- 直近実行（2026-05-07）：716問中 0件。
- **理由：** 著作権に対する明示的な防御線。

## 4. コンテンツの規模と品質保証

### 4.1 規模（v1.12.50 時点）

| 領域 | 件数 | 特記事項 |
|---|---|---|
| 文法パターン | 178 | 各パターンに ≥4 の例文・よくある誤用・ヒンディー語 L1 干渉ノート（`l1_notes.hi`） |
| 語彙 | 1,041 | 全件にヒンディー語訳（`gloss_hi`）、134 動詞に動詞種別（五段／一段／不規則）、6 件に group1 例外フラグ、87 名詞に助数詞、高頻度語にアクセント・コロケーション |
| 漢字 | 106 | 全件にヒンディー語意味（`meanings_hi`）、部首分解、記憶補助、字形類似クラスター、≥3 例 |
| 読解 | 45 文 | 全件にヒンディー語要約（`summary_hi`）、日本固有文化の説明（`cultural_context`） |
| 聴解 | 47 問 | VOICEVOX 0.25.2 多声収録（4声優：四国めたん／春日部つむぎ／白上虎太郎／青山龍星）、JLPT N5 速度帯 180–240 モーラ／分、全件にヒンディー語解説（`explanation_hi`） |
| 模擬試験 | 29 紙 426 問 | 文字／語彙／文法／読解＋聴解バーチャル試験紙＋実試験形式準拠の総合模擬（85問×105分） |

### 4.2 品質保証パイプライン

- **48 のコンテンツ整合性インバリアント（不変条件）** が CI で
  毎コミット時に自動チェック（`tools/check_content_integrity.py`）。
  代表的なもの：
  - JA-13：N5 範囲外漢字を含まないか
  - JA-15：すべての音声マニフェスト参照が実ファイルに解決するか
  - JA-30：過去問署名の混入無し
  - JA-37：英語フィールドがある項目はヒンディー語フィールドも
    必ず存在
  - JA-39：ロケール集合のロック（`{en, hi}` のみ）
- **8 の Muji-flat デザインシステム規則** （色・タイポ・余白等）
  も CI でチェック。
- **9 ラウンドの監査** が完了済み（rounds 1–9）。トラッカーは
  `feedback/n5-audit-2026-05-04.xlsx`（219 完了 / 3 不採用 /
  0 未着手）。
- **LLM-人格 Q33 品質基準** で全コンテンツが native_reviewed
  状態。これは「日本語教師の人格を与えた LLM による精査」で、
  人手による native レビューの代替として位置づけ、十分性の
  事後検証は **本ドキュメントの読者であるあなた** にお願い
  したい点です（→ § 5）。

## 5. 私たちが日本語母語話者レビュアーに伺いたいこと

本アプリは LLM-人格による品質管理を通過していますが、それが
真に「日本語教師の眼を通した」と言えるかは、**実際の日本語
教育者にレビューしていただいて初めて検証できます**。お願い
したい論点：

### 5.1 Critical（重要）論点

1. **文法パターンの J 妥当性** ：178 パターンの各 `pattern`／
   `meaning_ja`／`explanation_ja`／例文が JLPT N5 出題範囲・
   N5 学習者にとっての適切な理解と一致しているか。
2. **例文の自然さ** ：合計 ~700+ の例文が「教科書日本語」と
   して通用するか。違和感のある言い回し・古めかしい用法・
   存在しない用例が無いか。
3. **聴解スクリプトの自然さ** ：47問のスクリプトが JLPT 形式
   に合致し、N5 学習者にとって聴き取り可能な範囲に留まって
   いるか。
4. **読解文の自然さ** ：45 文が N5 範囲の漢字・語彙のみで構成
   され、かつ文章として不自然でないか。
5. **N5 範囲の境界判断** ：本アプリは JEES 公開ガイドラインを
   元に N5 範囲を判定していますが、境界事例（例：あした vs
   あす、わたし vs わたくし、いくら vs どのくらい）の扱いが
   日本語教育の現場感覚と一致するか。

### 5.2 Important（重要だが急がない）論点

6. **ヒンディー語ノート（`l1_notes.hi`）の妥当性** ：日本語
   教師としての観察に基づき、L1 ヒンディー語話者が抱える
   学習困難点を 8 種類に分類していますが（語順・助詞対応・
   性数一致・時制過剰標示・敬語階層・否定形・疑問詞位置・
   複数標示）、対象学習者の実際の誤用傾向と合致しているか。
7. **聴解音声の自然さ** ：合成音声（VOICEVOX）でも JLPT 受験
   訓練として通用するか。具体的なずれ（イントネーション、
   ポーズ、読み間違い）があれば指摘を歓迎します。後の肉声
   差し替えバッチに反映します（[`RECORDING-BRIEF.ja.md`](RECORDING-BRIEF.ja.md)
   参照）。
8. **ヒンディー語訳の品質** ：全コンテンツ100%カバー（
   `gloss_hi` / `meaning_hi` / `meanings_hi` / `summary_hi` /
   `explanation_hi`）はヒンディー語側の native レビューを
   想定しており、こちらは別途インド側パートナーに依頼予定
   です。本ドキュメントの読者には主に **日本語側の** 評価を
   お願いしています。

### 5.3 Nice-to-have（時間あれば）

9. **教育的順序付け** ：178 パターンの提示順、語彙の頻度順
   (rank-of-day-1 vs rank-of-day-100)、漢字の課順（`lesson_order`
   フィールド）が現場の学習進度と合致するか。
10. **N4 連続性** ：将来 N4 アプリを派生させた際、N5 内容と
    の橋渡しとして欠落しているトピックは無いか。

## 6. レビューしないでよい範囲

以下は **本アプリの設計上の選択** であり、賛否が分かれうる
ものですが、選択そのものは fixed です。論点としてはお受け
できますが、変更前提のフィードバックは設計原則に反するため
採用できません：

- アカウント機能を追加すること（§ 3.1 に反する）。
- 解析ツール・トラッキングを追加すること（§ 3.2 に反する）。
- 広告・有料機能を追加すること（§ 3.3 に反する）。
- ゲーミフィケーションを強化すること（§ 3.4 に反する）。
- JLPT 過去問テキストを取り入れること（§ 3.5 に反する）。
- N5 の範囲を超える内容を追加すること（プロダクトの境界）。
- アプリ UI を日本語化すること（学習者は日本語を読めない
  前提のため、UI は英語＋ヒンディー語に限定）。
