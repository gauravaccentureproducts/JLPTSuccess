# 日本語母語話者レビュアーパック

> **対象：** [`PROJECT-OVERVIEW.ja.md`](PROJECT-OVERVIEW.ja.md) を
> 読了済みで、JLPTSuccess の日本語コンテンツを評価いただける
> 日本語母語話者・日本語教育者。
> **作成：** 2026-05-07（ラウンド9 終了時、v1.12.50）。
> **作業時間目安：** 全サーフェスを通すと 8〜12 時間。サーフェス
> 単位で分割可能（→ § 1）。

---

## 1. レビューの分割と優先度

サーフェス単位で分担できます。優先度の高い順に：

| 優先度 | サーフェス | 件数 | 想定時間 | レビュー対象ファイル |
|---|---|---|---|---|
| ★★★ | 文法パターン | 178 | 4〜6時間 | `data/grammar.json` |
| ★★★ | 聴解スクリプト | 47 | 1〜2時間 | `data/listening.json` |
| ★★ | 読解文 | 45 | 1〜2時間 | `data/reading.json` |
| ★★ | 模擬試験 文法・読解部 | 426 | 2〜3時間 | `data/papers/*/paper-*.json` |
| ★ | 語彙 | 1,041 | 2〜3時間（速読可） | `data/vocab.json` |
| ★ | 漢字 | 106 | 1時間 | `data/kanji.json` |

「どの順序でも構いません」とは申しませんが、**文法と聴解** が
JLPT 受験に直接効く部分なので、時間が限られる場合はこの2つ
からお願いします。

## 3. サーフェス別チェックリスト

### 3.1 文法パターン（`data/grammar.json`）

各エントリは以下の主要フィールドを持ちます：

```json
{
  "id": "n5-001",
  "pattern": "〜は です／だ",
  "meaning_ja": "Aは Bです（A は B である）",
  "meaning_en": "A is B / A = B",
  "explanation_ja": "...",
  "explanation_en": "...",
  "examples": [
    {"ja": "私は 学生です。", "en": "I am a student.", "translation_en": "..."}
  ],
  "common_mistakes": ["..."],
  "review_status": "native_reviewed",
  "l1_notes": { "hi": "..." }
}
```

**確認ポイント：**

- [ ] **`pattern` フィールド**：JLPT N5 出題範囲に含まれるか。
      範囲外（N4 以上）の場合フラグ。
- [ ] **`meaning_ja`**：日本語学習者向けの意味記述として
      適切か。例えば「〜は です／だ」を「A は B である」と
      説明するのは N5 学習者には抽象的過ぎないか。
- [ ] **`examples`**：≥4 例文の各 `ja` フィールドが：
  - 自然な日本語か（「教科書日本語」として）
  - N5 範囲の漢字・語彙のみで構成されているか
  - パターンを明確に示しているか（パターンと無関係な複雑
    構造になっていないか）
  - 古めかしくないか（昭和的言い回しを避ける）
- [ ] **`common_mistakes`**：学習現場で実際に観察される
      誤用と一致するか。LLM 推測は実誤用と乖離することが
      あります。
- [ ] **`l1_notes.hi`**：ヒンディー語側 L1 干渉について
      日本語教師としての観察と一致するか（インド人学習者に
      会ったことがある場合のみ）。

### 3.2 聴解スクリプト（`data/listening.json`）

各エントリ：

```json
{
  "id": "n5.listen.001",
  "mondai": 1,
  "format_type": "task_understanding",
  "title_ja": "どこで 会いますか",
  "script_ja": "男の人と 女の人が...",
  "lines": [{"text_ja": "...", "startMs": 0}, ...],
  "prompt_ja": "あした、二人は どこで 会いますか。",
  "choices": ["...", "..."],
  "correctAnswer": "...",
  "explanation_ja": "...",
  "audio": "audio/listening/n5.listen.001.mp3"
}
```

**確認ポイント：**

- [ ] **JLPT 形式の妥当性**：mondai 1〜4 の各形式が公式
      JLPT N5 聴解問題と構造的に合致しているか。
  - mondai 1：課題理解（次に何をすべきか／何を買うか／
    どこへ行くか）
  - mondai 2：ポイント理解（理由・詳細・どうして・どこ）
  - mondai 3：発話表現（場面提示＋3候補発話）
  - mondai 4：即時応答（刺激＋3応答候補）
- [ ] **スクリプトの自然さ**：教科書日本語として通用するか。
      不自然な言い回し・存在しない表現が無いか。
- [ ] **N5 範囲制約**：N4 以上の文法・語彙・漢字が混入して
      いないか（漢字の混入は CI が止めますが、文法・語彙は
      人手判断要）。
- [ ] **正解の明確性**：`correctAnswer` がスクリプトから
      論理的に一意に導けるか。曖昧な場合フラグ。
- [ ] **distractor の妥当性**：誤答選択肢が「もっともらしい
      が間違い」になっているか。明らかに不自然・無関係な
      選択肢は除外候補。
- [ ] **音声の自然さ**（再生して確認）：合成音声 VOICEVOX
      の制約は理解した上で、明らかに不自然なポーズ・読み
      間違い・抑揚があれば指摘。後の肉声差し替えで反映
      可能。

### 3.3 読解文（`data/reading.json`）

各エントリ：

```json
{
  "id": "n5.read.001",
  "title_ja": "...",
  "ja": "...",
  "tier": "core_n5",
  "questions": [...],
  "summary_hi": "..."
}
```

**確認ポイント：**

- [ ] **語数・難度**：N5 学習者が辞書なしで読める範囲か。
      JLPT N5 読解の典型的長さ（80〜200字程度）に収まって
      いるか。
- [ ] **N5 範囲漢字制約**：N4 以上の漢字が混入していないか
      （CI が止めますが、再確認要）。
- [ ] **文章の自然さ**：簡潔だが不自然でない日本語か。
      初級教科書的すぎず、生硬すぎず。
- [ ] **comprehension question の妥当性**：本文から答えら
      れるか。本文に書かれていない推論を要求していないか。
- [ ] **`tier` フィールド**：`core_n5`（基礎）vs
      `late_n5`（後半）の分類が現場感覚と合致するか。

### 3.4 模擬試験紙（`data/papers/*/paper-*.json`）

29 紙、合計 426 問。文字（moji）・語彙（goi）・文法（bunpou）・
読解（dokkai）・聴解（chokai virtual）に分かれます。

**確認ポイント：**

- [ ] **問題形式の妥当性**：JLPT N5 公式形式に準拠している
      か（漢字読み、表記、文脈規定、言い換え、用法、文法、
      文章組み立て、文章の文法、内容理解、情報検索、聴解の
      4 mondai）。
- [ ] **distractor 設計**：誤答選択肢が「もっともらしい誤
      用」を反映しているか。受験者の典型的誤答パターン
      （送り仮名間違い、訓読み・音読み混同、類義語混同等）
      を反映しているか。
- [ ] **正解の一意性**：複数の解釈が成り立たないか（特に
      文法問題）。

### 3.5 語彙（`data/vocab.json`）速読可

1,041 件。全件チェックは現実的でないので、サンプリング推奨：

- [ ] **意味記述（`meaning_ja`／`gloss_en`）の正確性**：
      よく見るチェック対象は同音異義語、複数義の語、敬語
      バリエーション。
- [ ] **動詞種別（`verb_class`）**：134 動詞のうちサンプル
      で 5〜10 を抽出し、五段／一段／不規則の分類が正しい
      か。**特に `group1_exception` フラグの 6 件**
      （いる、おる、はいる、かえる、しゃべる、はしる 等が
      候補）が現場知識と一致するか。
- [ ] **助数詞（`counter`）**：87 名詞に付与されています。
      「本」が「冊」を取るか「本」を取るかなど、対象選択
      が適切か。
- [ ] **アクセント・コロケーション**：高頻度語のみに付与
      （カバレッジは限定的）。誤りがあれば指摘歓迎。

### 3.6 漢字（`data/kanji.json`）

106 字（N5 範囲完全カバー）。

**確認ポイント：**

- [ ] **音読み・訓読みリスト**：N5 学習者が覚えるべき範囲
      に絞られているか。マイナーな読みが混入していないか
      （例：「会」の え 読みは N5 では不要だが、いん 読み
      は不要、など）。
- [ ] **意味（`meanings_ja`／`meanings_en`）**：N5 文脈で
      の主要意味を捉えているか。
- [ ] **部首分解（`radical_breakdown`）**：日本語教育で
      一般的な分解と一致するか。教育的に有効な記憶補助か。
- [ ] **記憶補助（`mnemonics`）**：日本語学習者が学習時に
      使えるレベルか。LLM 生成の場合、現場感覚と乖離する
      ことがあります。
- [ ] **字形類似クラスター（`look_alike`）**：8 クラスター
      （大／犬／太, 木／本／末／未, など）が学習者の典型
      混同パターンと一致するか。
- [ ] **例語（`examples`）**：≥3 例の各語が：
  - N5 範囲漢字のみで構成されている、または対象漢字が
    保持され、他の N5 範囲外漢字はかな置換されている
  - N5 学習者にとって実用的な語か

## 4. 指摘ログのテンプレート

指摘の記録は CSV でも JSON でも構いません。GitHub Issue でも
受け付けます。フォーマット：

```csv
finding_id,surface,item_id,severity,category,description,suggested_fix
F-001,grammar,n5-115,HIGH,unnatural_example,"例文 [3] の 「コーヒーは 飲みますが」 が文脈不足で不自然","「あさ、コーヒーは 飲みますが、ばんは 飲みません」 のように時間枠を加えると自然"
F-002,listening,n5.listen.014,MEDIUM,N4_grammar,"スクリプト中の 「〜ようと思います」 は N4","「〜たいです」 に言い換え可能"
```

`severity` の目安：

- **CRITICAL**：JLPT 学習者にとって誤った日本語を教えて
  しまうレベル。リリース停止候補。
- **HIGH**：明らかな違和感。次のリリースまでに修正したい。
- **MEDIUM**：教育的にはやや劣るが致命的でない。
- **LOW**：好みのレベル・代替案の提示。

`category` 例：
- `factual_error`（事実誤認・誤用）
- `unnatural_example`（不自然な例文）
- `register_mismatch`（待遇表現のミスマッチ）
- `N4_or_above`（N5 範囲外混入）
- `pronunciation_ruling`（読み方裁定の誤り）
- `JLPT_format_drift`（JLPT 形式からの逸脱）
- `cultural_context`（日本固有事情の説明不足）
- `pedagogical_ordering`（順序付けの問題）

## 5. よくある質問

### Q1. レビューに対価はあるか？

[`PROJECT-OVERVIEW.ja.md`](PROJECT-OVERVIEW.ja.md) § 9 をご参照
ください。経済的対価はお支払いできません。代わりに
`CONTRIBUTORS.md` への記名（許可いただいた場合）と CHANGELOG
記載で謝意を表します。

### Q2. 指摘した事項はいつ反映されるか？

GitHub Issue で受領後、次の audit round（四半期に1回程度
予定）で集中的に対応します。CRITICAL 事項は緊急修正リリース
で個別対応する場合があります。

### Q3. すべて同意しないといけないか？

いいえ。指摘内容について「採用しない」判断もありえます
（特に [`PROJECT-OVERVIEW.ja.md`](PROJECT-OVERVIEW.ja.md) § 6
の設計上の選択に関わる場合）。ただし採否は CHANGELOG または
GitHub Issue 上で公開します。

### Q4. レビュー範囲は完全に分担できるか？

はい。サーフェス単位（文法／聴解／読解／模擬／語彙／漢字）
での分担が可能です。全部を一人でレビューする必要は
ありません。

### Q5. ローカルで動かさず、オンライン版だけでレビュー
してよいか？

はい。オンライン版で全コンテンツに到達できます。
ローカル版は JSON 直接閲覧したい場合のみ必要です。

### Q6. 指摘事項の例を見たい

[`feedback/closed/native-teacher-review-request.md`](../feedback/closed/native-teacher-review-request.md)
（過去の英語版レビュー依頼）に過去ラウンドの指摘例があります。
日本語側の指摘は本パックが初めての公式依頼です。
