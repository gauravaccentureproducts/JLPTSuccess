# 録音ブリーフ（声優・収録エンジニア向け）

> **作成者：** JLPTSuccess プロジェクト常駐 日本語教師（ラウンド9 締切、2026-05-07）
> **対象：** JLPTSuccess 聴解音声差し替え（IMP-094）の録音セッションを担当する声優および収録エンジニア
> **目的：** 教育的判断をスタジオで即興でしないで済むよう、収録準備一式をあらかじめお渡しする

このブリーフは IMP-094（合成音声から肉声への差し替え）の **収録前**
成果物です。本ブリーフ自体は音声を生成しません。将来予算または
ボランティア声優が確保された時点で、即セッションを始められるよう
事前準備した資料です。

英語版ブリーフは [`RECORDING-BRIEF.md`](RECORDING-BRIEF.md) です。
内容は本ファイルと同一です（言語のみ違います）。

## 1. 声の方向性（アナウンサー中立、JLPT 忠実度）

JLPT 公式の聴解音声は **NHK アナウンサー風の標準語** が基準です。
具体的には：

| 項目 | 仕様 | 補足 |
|---|---|---|
| 方言 | 東京標準語（共通語） | 関西弁・東北弁などは不可（JLPT 聴解問題として無効になります） |
| 待遇表現 | 原則「です・ます」体。スクリプトが「だ・る」を指定する時のみカジュアル | スクリプトの体に合わせてください |
| トーン | 落ち着いた中立トーン。**感情的色付けは避けてください** | 〜たい文を「楽しそうに」読む、〜なければなりません文を「ストレス気味に」読む等は **正解を漏らす** ので不可 |
| 速度 | **180–240 モーラ／分**（JLPT N5 目標帯） | 自然会話の約 300+ より遅く。1モーラあたり 250–333 ms 程度。合成音声 VOICEVOX ベースラインは「四国めたん」を `speed_scale=1.30` でこの帯に乗せています |
| ポーズの扱い | **スクリプトに書かれたポーズ通り。** 即興で読点や息継ぎを増やさないでください | 文中ポーズは N5 学習者の聴解難易度を変えてしまいます。合成版で起きた「文中の読点で音声が切れる」バグは、TTS が `、` で硬い停止を入れたのが原因でした |
| マイク距離 | 一定。話者ごとにマイク近接が変わらないこと | N5 では聴き手が話者役を声よりもテキストの「男:／女:／先生:／学生:」で識別しますが、レベル差は理解を阻害します |

### 速度の目安

JLPT N5 聴解問題は通常 **120–250 モーラ** です。目標速度では
**1問あたり 30 秒〜1分20秒** の音声長になります。

| 問題形式 | モーラ数の目安 | 目標音声長 |
|---|---|---|
| 問題1 課題理解（対話） | 100–180 モーラ | 35〜60 秒 |
| 問題2 ポイント理解（対話） | 120–200 モーラ | 40〜70 秒 |
| 問題3 発話表現（一文） | 8〜15 モーラ | 5〜10 秒 |
| 問題4 即時応答（一文） | 5〜10 モーラ | 3〜6 秒 |

10% 以上ずれる場合は速度を調整してください。ただし **ポーズ編集
で帳尻合わせはしないでください**。文単位のテイクをそのまま使用
してください。

## 2. 問題形式別の演出指示

JLPT N5 聴解の4形式はそれぞれ異なる音響プロファイルです。形式に
合わせて読み分けてください。

### 問題1 — 課題理解

短い対話を聴き、登場人物が次に何をすべきか・どこで会うか・何を買うか
等を判断する形式。

- **2人の話者が必要。** スクリプトが同性役（兄弟・母娘等）を指定
  する場合を除き、男女各1名を基本とします。
- **ターン間ポーズ：** 約 250 ms（自然会話よりやや長め——試験は
  流暢さより明瞭さを重視します）。
- **ナレーション冒頭：** 通常1行目はナレーターが場面設定を読みます
  （`男の人と女の人が話しています…`）。**第三者アナウンサー声で、
  対話より平板に**読んでください。キャラクター付けは不可。
- **質問文：** ナレーターの締めくくり（`二人はどこで会いますか`）が
  解答対象です。**対話より一段ゆっくり、はっきりと**。疑問詞
  （どこ／何／いつ／だれ／どうして）にわずかな韻律ピーク。

### 問題2 — ポイント理解

問題1と同様の形式ですが、質問が特定の理由・詳細（なぜそうなったか・
何を買ったか等）を訊きます。

- 2人話者ルールは同じ。
- ナレーション冒頭・質問文の規律も同じ。
- **ダミー情報の行**（事実だが解答ではない情報）は韻律強調しない
  でください。強調は学習者にとって「これが解答」のサインに
  なります。事実情報の行はすべて同じ中立レジスタで。

### 問題3 — 発話表現

場面設定の一文（`～のとき、何と言いますか`）に続いて、3つの短い
候補発話。聴き手は最も自然な発話を選びます。

- **場面設定はナレーターのみ** — 中立、やや遅め、キャラ付け無し。
- **3つの候補発話**は1つずつ別の話し手で、場面に合うレジスタで
  読んでください（フォーマルな初対面の人 → 丁寧な大人男女、同年代の
  友達 → カジュアル、子供 → 若い声優が居れば若めに）。
- **正解を漏らす感情的合図は不可。** 不正解候補も正解候補と同じく
  自然に聴こえる必要があります。試験対象は **言葉** であって
  トーンではありません。

### 問題4 — 即時応答

刺激の一文（質問・あいさつ・発言）に続いて、3つの短い応答候補。

- **刺激文** は1名、**応答候補** は理想的には別の1名で読み分けて
  ください。話者交代を内容前から音響的に明示できます（N5学習者
  はこの音響合図に頼ります）。
- 応答候補3つはすべて **同じくらい社会的に妥当に** 聴こえなければ
  なりません。再びトーンでの示唆は不可。
- 短い問題（合計3〜5秒）が多いです。ワンテイクでクリーンに、
  スプライス無しで。

## 3. 読みの裁定（標準化）

N5 コーパスには **複数の正当な読みを持つ語** が含まれます。
オンスクリーンのルビと音声を完全一致させるため、本節の裁定に
従ってください。声優の直感と異なる場合は **収録前に** 制作側へ
フラグを立てて確認をお願いします。無断置き換えは不可。

### 3.4 アクセント裁定（弁別が必要な箇所）

JLPT N5 は直接的にはアクセントを問いませんが、ホモグラフを
弁別するために NHK アクセント辞典の標準形に従ってください：

| 表記 | 意味 | アクセント（東京式） | 型 |
|---|---|---|---|
| 雨 | あめ（rain） | あ↓め | 頭高 |
| 飴 | あめ（candy） | あめ↑ | 平板 |
| 橋 | はし（bridge） | は↑し↓ | 尾高 |
| 箸 | はし（chopsticks） | は↓し | 頭高 |
| 端 | はし（edge） | は↑し | 平板 |
| 紙 | かみ（paper） | か↑み | 平板 |
| 髪 | かみ（hair） | か↑み | 平板（同型——文脈で弁別） |
| 神 | かみ（god） | か↓み | 頭高 |
| 今 | いま（now） | い↓ま | 頭高 |
| 居間 | いま（living room） | い↑ま | 平板 |

N5 コーパスは混同コンテキストでこれらを避けていますが、
例えば文頭に 雨 が出た場合は 頭高（第2モーラで降下）で
読んでください。

### 3.5 外来語（カタカナ語）

英語由来の外来語は **日本語の** アクセント・モーラ構造で読みます。
英語化しすぎないでください。N5 コーパスの例：

| 表記 | 読み | アクセントヒント |
|---|---|---|
| コーヒー | こ↓ーひー | 頭高（長音 ー で降下） |
| テレビ | て↑れび | 平板 |
| ラジオ | ら↑じお | 平板 |
| カメラ | か↓めら | 頭高 |
| パン | パ↓ン | 頭高 |
| ノート | の↑ーと | 平板 |
| ペン | ペ↑ン | 平板 |
| デパート | で↑ぱーと↓ | 中高 |
| アパート | ア↑ぱーと↓ | 中高 |
| エレベーター | え↑れべーたー↓ | 中高 |
| バス | バ↑ス | 平板 |
| タクシー | タ↑くしー | 平板 |

**重要：** 長音 `ー` は1モーラとしてフルに数えてください。
`コーヒー` は **4モーラ**（こ・ー・ひ・ー）であって2モーラでは
ありません。

## 4. 収録セッションの手順

### 機材

- **マイク：** 大口径コンデンサー、カーディオイド指向性。整音
  済みのブース。日本語の促音および p/t/k 系の破裂が目立つので
  ポップフィルター必須。
- **サンプリング：** 最低 44.1 kHz / 16bit。理想 48 kHz / 24bit。
- **納品形式：** モノラル WAV（生テイク）→ スタジオで正規化し、
  問題ごとに MP3 へ。

### 問題ごとの納品仕様

各 `n5.listen.NNN` ID について：

1. **モノラル MP3 1ファイル**、128 kbps CBR、**−16 LUFS**
   インテグレーテッド、ピーク −1 dBTP 以下。
2. **ファイル名**：プロジェクト既存パスと一致：
   `audio/listening/n5.listen.NNN.mp3`。マニフェスト・
   `<audio src>` 参照を変更不要にするため。
3. **長さメタデータ：** プロジェクトの
   `tools/audit_audio_coverage.py` および
   `tools/fix_truncated_audio_2026_05_07.py` がスクリプトの
   モーラ数と音声長を比較します。健全な納品テイクは JLPT N5
   目標帯 180–240 モーラ／分の範囲内のはず。範囲外の場合、
   切り詰め検出ツールがフラグを立てます。

### テイク方針

- **問題は1テイクで通して読んでください。** 行ごとのスプライスは
  避けてください。1テイクの自然なリズムと息継ぎが教育的シグナル
  の一部です。
- **問題ごとに最低2つのクリーンテイク。** ポストでよい方を
  選び、別テイクは保管。
- **アウトテイク・息編集は不可。** 前無音 ≤ 200 ms、後無音 ≤
  300 ms。これを越えるとアプリの自動進行ロジックが混乱します。
- **コーパス全体で同じ声優を使う**のが望ましいです。男女2名を
  使う場合、47問すべて同じ2名で。途中での声優交代は学習者の
  「同じ話者か？」判断を狂わせます。

### 読みのフラグ立て

ある語の読みについて疑問がある場合：

1. **テイクログにフラグ。** 無断で置き換えないこと。
2. 本ブリーフ作者（プロジェクト常駐 日本語教師）が裁定して
   から最終納品します。

### 行番号・タイムスタンプの整合性

問題1・2 の複数行対話には、`data/listening.json` に行ごとの
`startMs` タイムスタンプが入った `lines[]` 配列があります。
納品音声の実際の行境界は、それらタイムスタンプから ±300 ms
以内に収まるべきです。アプリの **トランスクリプト同期再生**
が正しく動くために必要です。

プロジェクト側で `tools/listening_align_2026_05_07.py`（実装
予定）を納品時に再実行し、実音声から `startMs` を再スタンプ
します。微小なずれは許容。1秒以上のずれは行抜け・スプライス
の疑い。

## 5. ディレクション付きサンプル問題

ID `n5.listen.001`、`005`、`009`、`041`（各問題形式から1問）に
加え、`004`、`008`、`028`、`045`（同じく各形式の2問目）の
合計8問について、項目別の収録ディレクションシートを
**`data/recording_directions.json`** に用意しました。これらを
**パイロットバッチ** として収録方針を全関係者で擦り合わせ、
合意に達したあと残り39問は本ブリーフの規約だけで進めてください
（問題ごとの追加指示は不要になります）。

> **なぜ別ファイルか：** ディレクションには 韻律・中立 など
> 技術語彙が含まれます。これらは **学習者向け** には N5 範囲外
> ですが、**録音エンジニア向け** には正当な語彙です。
> `data/listening.json` から分離することで、JA-13（N5 範囲外
> 漢字を許さない不変条件）に引っかからずに、本番側スキーマも

## 6. 受け入れ基準

納品バッチの受け入れ条件：

- [ ] すべての問題の音声長が問題形式ごとの目標帯（§ 1 表）に
      入っている。
- [ ] `tools/audit_audio_coverage.py` が 100% カバレッジを
      報告する。
- [ ] `tools/fix_truncated_audio_2026_05_07.py` が0件の切り
      詰めをフラグする。
- [ ] 母語話者によるスポットチェック（10% ランダム抽出）
      で：読み裁定が遵守されている、方言が混入していない、
      ダミー選択肢に感情的合図が無い、ことを確認。
- [ ] `tools/check_content_integrity.py` が exit 0
      （JA-15 missing-file 不変条件違反なし）。
- [ ] 差し替え済み問題に `voice: "native"` が
      `data/audio_manifest.json` および
      `data/audio_manifest_voice.json` で設定されている
      （後続の合成パイプラインで再生成されないため）。
- [ ] `sw.js` の `CACHE_VERSION` バンプ済み（既訪問者が新音声
      を取得するため）。

| 文法（631 例文） | 631 × 約3秒 | 約 2–4 時間 | USD$200–800 | USD$200–400 | **USD$400–1200** |
| 聴解（47 問） | 47 × 約30秒 | 約 2–4 時間 | USD$200–800 | USD$160–320 | **USD$360–1120** |

**最初のバッチは読解のみ推奨** — 現行合成ベースラインに対する
品質向上幅が最大、予算最少。

## 8. 本ブリーフの対象外

- 語彙（`data/vocab.json`）の単語別収録。Forvo 形式の単語音声
  は別の延期サーフェス。
- 文法例文を聴解と同じ声優で収録すること。ドメインが異なる
  （朗読型例文 vs 対話型）ので、予算が許せば声優を分けます。
- 既存の合成 VOICEVOX 聴解音声の再収録。読解バッチの結果次第
  で再検討。
